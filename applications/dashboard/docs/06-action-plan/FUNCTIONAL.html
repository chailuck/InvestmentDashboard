Action Plan — Functional Specification

Overview

The Action Plan section lets traders prepare and save structured trading plans before executing them in their brokerage account. Two independent plan types serve different planning stages:

TypePurposeData source
Purchase Action PlanStocks not currently in the portfolio that the trader intends to buyFree entry — user types each symbol
Portfolio Action PlanActions on currently open positions (adjust TP, set SL, plan order sizes)Live portfolio data from the Excel file

1. Navigation

The Action Plan item in the main sidebar leads to the plan list page (/action-plan). This page renders two collapsible sections — Purchase Plans and Portfolio Plans — each with its own table, filter controls, and Create button.

2. Purchase action plan

2.1 Plan list

The Purchase Plans section shows a paginated list of saved plans with the following controls:

  • Create new plan button — opens a name-entry modal
  • View selector — filter by time period: 3 months (default) · 6 months · 1 year · All
ColumnDescription
CreatedDate-time the plan was first saved
Plan NameHuman-readable name (e.g. 2026-06-01)
SymbolsComma-separated list of stock tickers, truncated after 7
ActionsEdit (pencil icon) · Delete (trash icon, with confirmation) · Duplicate (copy icon)
Last ModifiedDate-time of last save

Create / Duplicate flow: A modal pre-fills the name with today's date (falling back to YYYY-MM-DD-01, -02, etc. if names are already taken). The user may change the name before confirming. On confirm, the backend creates the plan header and the browser navigates directly to the plan editor.

2.2 Purchase plan editor

URL: /action-plan/purchase/[id]

An editable spreadsheet-style table where each row represents one stock the trader plans to buy. Rows can be added with the + Add Row button and reordered by drag.

ColumnEditableNotes

Strategy options

  • BREAK OUT
  • BUY ON DIP
  • แท่งเทียนกลับตัว (Candle Reversal)
  • NEWS
  • AJ PAO
  • OTHERS (reveals a free-text input field)

RR (risk/reward) indicator

The RR column shows a visual quality indicator for each row's risk/reward setup:

  • RR ≥ 3 — green check mark: excellent risk/reward ratio
  • 1 ≤ RR < 3 — amber warning: acceptable but not ideal
  • RR < 1 — red cross: poor risk/reward; reconsider the plan
  • Blank / invalid (e.g. BUY = SL) — no indicator shown

Toolbar actions

  • + Add Row — appends a new blank row at the bottom
  • Refresh Prices — fetches the latest live price for every stock row simultaneously and updates the CURRENT PRICE column in-memory (see below)
  • Save — persists all rows to the database (full replace: delete all + bulk insert)
  • Generate — opens a read-only modal with the plan in a copy-ready text format for sharing with a broker or tracking system

Refresh Prices button

The Refresh Prices button (RefreshCw icon) appears in the toolbar between Generate and Save. It re-fetches live current prices from Yahoo Finance for all rows that have a stock symbol, in parallel, and updates the CURRENT PRICE cells in-memory without saving to the database.

During refresh:

  • The button label changes to Refreshing... and a spinner is displayed.
  • Both the Refresh Prices button and the Save button are disabled for the duration of the refresh.
  • Rows whose price fails to load retain their previous CURRENT PRICE value.

After refresh, an inline notification is shown below the toolbar:

OutcomeNotification messageStyle
All prices fetched successfullyRefreshed N price(s) successfully.Success
Some prices failedRefreshed N of M prices. Failed: SYMBOL1, SYMBOL2 (and K more).Partial
All prices failedFailed to refresh all N prices. Failed: SYMBOL1, SYMBOL2 (and K more).Failure
No symbols in the planNo symbols to refresh.Info

The notification includes a Dismiss (×) button. It does not auto-dismiss. A new refresh replaces the previous notification.

Refresh Prices does not save. The user must click Save separately to persist the updated prices to the database.

3. Portfolio action plan

3.1 Plan list

The layout and controls are identical to the Purchase Plan list section. The only difference is the plan type scoping — this section shows only plan_type = 'portfolio' plans.

3.2 Portfolio plan editor

URL: /action-plan/portfolio/[id]

Unlike the Purchase editor which is populated manually, the Portfolio editor loads live position data from the portfolio tracker and merges it with any saved planning data. On page load:

  1. Fetches the saved plan from the database (saved TP / SL / Order Size)
  2. Fetches all currently active positions from the Portfolio Tracker API (live Current Price, Size, Entry Price from Excel)
  3. Merges: position data populates the read-only columns; saved plan data overlays TP / SL / Order Size
ColumnEditableSource
SYMBOLNoPortfolio tracker (active positions)
CURRENT PRICENoLive price from Yahoo Finance via portfolio tracker
SIZENoPosition size from Excel file
ENTRY PRICENoEntry price from Excel file
TPYesUser input, persisted to database
SLYesUser input, persisted to database
ORDER SIZEYesShares to buy/sell in the planned action; persisted

Sidebar widget Refresh button: Appears in the collapsed Portfolio Action Plan sidebar widget (not the full editor). When clicked, the widget fetches the current plan directly from the server (bypassing any cached data), fetches live market prices for each symbol, writes the updated plan back to the server with only current_price overwritten, and then invalidates the React Query cache so the widget re-renders. All other plan fields (entry_price, sl, tp) are taken from the server's authoritative state, not from memory. See TECHNICAL.html — sidebar refresh fix for the implementation detail.

4. Business rules

  1. Plans are user-scoped — each user sees only their own plans. A user cannot access another user's plans even with a direct URL.
  2. Plan names must be unique within the same user + plan type combination. The suggest-name API handles this automatically by appending a suffix if today's name is already taken.
  3. Deleting a plan is irreversible. A confirmation modal ("Are you sure you want to delete plan name?") is required before deletion.
  4. Duplicating a plan creates an exact copy of all line items under a new name. Changes to the duplicate do not affect the original.
  5. The Portfolio Plan editor always loads live position data on every page open. The saved TP/SL/order_size values overlay the live data. Positions that have been closed since the plan was last saved will no longer appear in the live list.
  6. RR (risk/reward ratio) is display-only and is never stored in the database. It is recalculated client-side on every render from the current BUY, TP, and SL values.
