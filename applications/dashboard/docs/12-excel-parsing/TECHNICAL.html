Excel Parsing — Technical Specification

This document specifies exactly how the backend reads and interprets the portfolio Excel file, how live prices are fetched, how caching works across multiple workers, and how P&L is calculated. Source: backend/app/services/portfolio_excel.py.

1. File Path Configuration

Two path concepts exist: the source (read-only, on the host volume) and the working copy (writable, inside the container). The service always reads from the working copy.

ConceptPurposeDefault env varOverride (app_config.json key)
Source pathRead-only master file (Docker volume)INVESTMENT_EXCEL_SOURCE_PATH → INVESTMENT_EXCEL_PATHexcel_source_path
Working pathWritable copy the service reads fromINVESTMENT_EXCEL_PATHexcel_working_path
When a user triggers Refresh from Source in the UI, copy_excel_from_source() copies the source to the working path and busts all worker caches. The source file itself is never modified by the application.

1.1 Startup Auto-Copy

# _ensure_working_copy() — called before every _load_df()
if not working_path.exists():
    copy_excel_from_source()   # auto-copies on first access

2. Sheet Structure

2.1 Sheet Name

Hard-coded constant: SHEET = "Sheet1". The workbook must have exactly this sheet name. Any other sheets are ignored.

2.2 Required Columns

All column names are case-sensitive and must match exactly as shown.

Column nameType after parsingNotes
SymbolstringThai SET ticker (e.g. PTT, GULF) or DR code (e.g. AAPL01). Trailing whitespace is stripped with .strip().
Entry Datedatetime (coerced)pd.to_datetime(errors="coerce") — pandas accepts most date formats including Excel serial numbers, YYYY-MM-DD, DD/MM/YYYY. Unparseable values become NaT.
Exit Datedatetime (coerced)Same as Entry Date. NaT = position is still open.
Entry Pricefloat (coerced)pd.to_numeric(errors="coerce"). Non-numeric strings (formulas, text) become NaN.
Exit Pricefloat (coerced)Key field: NaN / blank = active/open position. Any numeric value = closed position. Formulas that return numeric values are resolved by Excel before pandas reads them. A formula returning the text string "NOT SELL" is treated as non-numeric → becomes NaN → position counted as active.
Current pricefloat (coerced)Optional cached price column. Used as fallback when Yahoo Finance is unavailable for an open position. Accessed via df.get("Current price", ...) so it is safe if the column is absent.
Position Sizeint (coerced then cast)Number of shares. Non-numeric → NaN → treated as 0.
Position (Long/Short)stringCase-insensitive. Direction multiplier: if "short" is in the lowercased value, P&L is inverted. Default "Long" if missing.
SLfloat (optional)Stop loss price. Accessed with .get("SL") — safe if column absent. NaN → None in output.
TPfloat (optional)Take profit price. Same as SL.
Remarksstring (optional)Free-text notes. NaN → None in output.
Column names are exact strings — including capitalisation and spaces. "entry date" or "EntryDate" will not be found and will raise a KeyError at runtime.

2.3 Active vs Closed Detection

# Active (open) position: Exit Price is NaN
open_mask = df["Exit Price"].isna()

# Closed position: Exit Price has a numeric value
closed_mask = df["Exit Price"].notna()

# Filtering by status
if status == "active":   df = df[open_mask]
if status == "closed":   df = df[~open_mask]
if status == "all":      df = df            # no filter

3. P&L Calculations

3.1 Net P&L

def _net_pnl(entry: float, exit_p: float, size: float, direction: str) -> float:
    mult = -1 if "short" in str(direction).lower() else 1
    return round((exit_p - entry) * size * mult, 0)

# Long example: entry=10.00, exit=12.00, size=1000
#   → (12 - 10) * 1000 * 1 = 2,000 ฿
# Short example: entry=10.00, exit=8.00, size=1000
#   → (8 - 10) * 1000 * -1 = 2,000 ฿

3.2 P&L Percentage

def _pnl_pct(entry: float, exit_p: float, direction: str) -> float:
    if entry == 0: return 0.0
    mult = -1 if "short" in str(direction).lower() else 1
    return round((exit_p - entry) / entry * 100 * mult, 2)

# Percentage is relative to entry price, not total investment

3.3 Open Position Current Price Fallback Chain

# For active positions, current price resolved in this order:
1. live_prices.get(symbol)         # Yahoo Finance (cached 5 min)
2. float(row["Current price"])     # Excel column if non-NaN
3. entry_price                     # last resort — no P&L shown

4.2 Request Strategy — Two-Level Fallback

Primary:  Yahoo Finance v7 batch API
          GET https://query1.finance.yahoo.com/v7/finance/quote
          ?symbols=PTT.BK,GULF.BK,...
          &fields=regularMarketPrice,regularMarketChangePercent,...

Fallback: Yahoo Finance v8 chart API (per-symbol, sequential)
          GET https://query1.finance.yahoo.com/v8/finance/chart/{symbol}
          ?interval=1d&range=5d
          (Used when v7 returns 401 or 429)

Last resort: yfinance.download() library (used in fetch_live_prices only)
             yf.download(symbols, period="5d", interval="1d")

Request headers sent to Yahoo Finance

User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 ...
Accept: application/json, text/plain, */*
Origin: https://finance.yahoo.com
Referer: https://finance.yahoo.com/
Timeout: 15 seconds (httpx.Client)
Yahoo Finance data for Thai SET stocks is typically 15–20 minutes delayed during market hours. Real-time data requires a paid data feed.

5.3 Multi-Worker Cache Invalidation

Uvicorn can run multiple worker processes. Each has its own _CACHE dict. A shared file at /app/uploads/.cache_bust acts as an invalidation signal.

# Validity check (runs on every cache lookup)
entry_valid = (
    entry exists
    AND (now - entry.timestamp) < ttl      # not expired
    AND entry.timestamp > bust_file.mtime   # written after last bust
)

# Bust trigger (called by copy_excel_from_source())
_CACHE_BUST_FILE.write_text(str(time.time()))
_CACHE.clear()   # also clears this worker's dict immediately
When running a single worker (default Docker Compose setup), the bust file mechanism is still used but provides no additional benefit. It becomes important only when --workers N is passed to Uvicorn.

6. Position Output Schema

The get_positions() function returns a list of dicts. This is the canonical shape used by all API endpoints and the AI copilot tool.

{
  "id":           int,           # row index + 1 (1-based)
  "symbol":       str,           # stripped ticker
  "direction":    str,           # "Long" or "Short" (raw cell value)
  "entryDate":    str | None,    # ISO 8601 date, e.g. "2025-11-03"
  "exitDate":     str | None,    # ISO 8601 date, None if open
  "entryPrice":   float,
  "currentPrice": float,         # live price (open) or exit price (closed)
  "exitPrice":    float | None,  # None if open position
  "positionSize": int,           # shares
  "netPnl":       float,         # rounded to 0 decimal places
  "pnlPct":       float,         # rounded to 2 decimal places
  "sl":           float | None,
  "tp":           float | None,
  "status":       "active" | "closed"
}

7. Performance History Output Schema

7.1 get_daily_performance() — period buckets

{
  "date":          str,    # sort key — "YYYY-MM-DD" (daily), "YYYY-MM-DD" (weekly Mon), "YYYY-MM" (monthly)
  "label":         str,    # display string — "03/06/2025" | "02/06/2025 – 06/06/2025" | "Jun 2025"
  "dailyPnl":      float,  # net P&L for this period (closed trades only)
  "cumulativePnl": float   # running total from earliest to this period
}

7.2 get_performance_by_stock() — per-symbol aggregation

{
  "symbol":       str,
  "net":          float,   # total realized + unrealized P&L
  "investment":   float,   # sum of (entryPrice * positionSize)
  "currentValue": float,   # sum of (currentPrice * positionSize)
  "pnlPct":       float,   # net / investment * 100 (0 if no investment)
  "wins":         int,
  "losses":       int,
  "total":        int,     # wins + losses
  "winRate":      float    # rounded to 1 decimal
}
Performance history only counts closed (realized) trades. Open positions are included in get_performance_by_stock() but are excluded from get_daily_performance().
