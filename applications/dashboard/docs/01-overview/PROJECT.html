InvestPro — Project Overview

Last updated: 2026-05-30

1. What Is InvestPro?

InvestPro is a full-stack, self-hosted investment dashboard designed for tracking a personal Thai SET (Stock Exchange of Thailand) portfolio. It provides real-time profit & loss calculations, interactive charting, AI-assisted trade analysis, and structured trade planning — all running inside Docker containers on a local machine.

Primary users: Individual traders managing a Thai SET portfolio who want a private, fully-controlled dashboard without relying on brokerage web portals or third-party cloud services. All data remains local and under the user's control.

The application requires no internet access for core portfolio features. An Anthropic API key is only needed for the AI Copilot chat feature.

2. Goals

GoalDescription
Portfolio visibilitySee all open positions, P&L, win/loss metrics at a glance on the dashboard
Performance analysisDaily, weekly, and monthly charts with drill-down to individual transactions by period or stock
Trade planningCreate and save structured purchase plans and portfolio action plans with risk/reward calculation
AI assistanceChat with an AI agent that has full context of the portfolio, live prices, and a personal knowledge base
Weekly scanStructured weekly review workflow with colour-coded evaluation, chart analysis, and direct integration with purchase plans
Self-hostedNo cloud dependency; all data stays local; works behind any tunnel (ngrok, Cloudflare Tunnel, etc.)

3. Non-Goals

  • Multi-user portfolio isolation — there is one shared Excel file as the source of truth for all users
  • Real-time WebSocket price streaming from the exchange (prices are polled on demand from Yahoo Finance)
  • Order execution or brokerage API integration
  • Mobile native app (the web UI is responsive but not optimised for mobile trading)
  • Email delivery of password reset tokens (tokens are currently returned in the API response for development use)

6. Quick Start

Prerequisites

  • Docker Desktop with Compose v2 (included in Docker Desktop 4.x and later)
  • The Excel file Investment tracking.xlsx accessible on the host machine
  • An Anthropic API key (only required for AI Copilot; all other features work without it)

Steps

# 1. Navigate to the application directory
cd applications/dashboard

# 2. Create the backend environment file from the example
cp backend/.env.example backend/.env

# 3. Edit backend/.env — set at minimum:
#    ANTHROPIC_API_KEY=sk-ant-...
#    ADMIN_EMAIL=admin@example.com
#    ADMIN_PASSWORD=ChangeMe123!

# 4. Update docker-compose.yml — change the host path for investment_data
#    to point to the directory containing your Excel file:
#    - /path/to/your/excel/dir:/app/investment_data:ro

# 5. Build all images and start all services in detached mode
docker compose up -d --build

# 6. Open the application in a browser
open http://localhost:3000
# Log in with the admin credentials you set in step 3
On first startup the backend initialises the database schema and seeds the admin account. This takes 15–30 seconds. The frontend service waits for the backend health check to pass before starting, so the app will not be available until the backend is fully up.

Minimum backend/.env

# Required for AI Copilot — get this from console.anthropic.com
ANTHROPIC_API_KEY=sk-ant-...

# First-run admin account (seeded once on startup via INSERT ... ON CONFLICT DO NOTHING)
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=ChangeMe123!
ADMIN_NAME=Admin

# Excel source file path inside the container
# The directory /app/investment_data is mapped from the host via docker-compose.yml
INVESTMENT_EXCEL_SOURCE_PATH=/app/investment_data/Investment tracking.xlsx
After logging in, go to Settings → App Configuration to verify or update the Excel file path if needed. You can also use the Test button there to confirm the path is readable.
