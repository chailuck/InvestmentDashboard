Analytics — Functional Specification

The Analytics page lets users search any financial symbol across Thai SET, Crypto, and Depositary Receipt markets, then view a full technical analysis modal with interactive charts, AI-generated analysis logs, Fibonacci charts, and personal notes.

1. Page Overview

Route/analytics
Auth requiredYes — any authenticated role
State persistenceSearch history lives in React useState — cleared on page navigation. No URL params, no server-side state.
Data sourcesYahoo Finance (live prices + OHLCV), investment agent knowledge base (analysis logs + Fibonacci charts), PostgreSQL (symbol notes)

2. Asset Type Selector

Three mutually-exclusive tabs control which market the search targets. The selected type determines how the backend resolves the ticker symbol.

The backend also accepts US, HK, and OTHER as asset types (used by the analytics service), but the Analytics page UI exposes only SET, CRYPTO, and DR tabs. The additional types are available via direct API calls.

3. Search Form

  • Single text input — values are automatically converted to UPPERCASE as the user types.
  • Input placeholder changes by asset type: "e.g. BBCP, AOT, KBANK" / "e.g. BTC, ETH, SOL" / "e.g. GOLD, OIL".
  • Submit via the Search button or pressing Enter.
  • The Search button is disabled while a request is in flight (spinner shown) or when the input is empty.
  • On success: the symbol is added to the result history table AND the Analytics Modal opens immediately.
  • On failure (symbol not found): an error banner appears below the form — "Symbol "X" not found on SET." The symbol is still added to history with found: false.
  • On network error: "Search failed. Check your connection."

4. Search Result History Table

Every successful search request is added to an in-page history table. The table persists as long as the user stays on the Analytics page.

Deduplication rule

If the same (symbol, assetType) pair is searched again, the previous entry is removed and the new one is inserted at the top. The list is capped at 20 entries.

5. Analytics Modal

A full-viewport overlay (80vw wide) with scrollable content. Clicking the backdrop or the × button dismisses it. Opening the modal triggers three parallel API calls: Analysis Log, Fibonacci Chart, and Symbol Note.

5.1 Modal Header

  • Symbol name (large, bold, monospace)
  • Asset type badge (coloured, matching the search tab)
  • Subtitle: "Candlestick · RSI · Stochastic · VRVP"
  • × close button

The main chart uses Apache ECharts with four panes rendered in a single canvas:

Interval selector

Four buttons above the chart switch the time interval. The selected interval is highlighted in brand colour.

ButtonInterval valueHistory windowNotes
1H1h~2 yearsFetches 1h bars from yfinance
4H4h~2 yearsFetches 1h from yfinance, resampled to 4h on the backend

Changing the interval triggers a new GET /analytics/chart request. A loading overlay is shown while fetching. The chart includes a dataZoom slider at the bottom for panning/zooming the time axis.

VRVP (Volume Range Visible Profile)

A horizontal bar chart overlaid on the right side of the price pane. It shows volume distribution across 24 equal price buckets spanning the entire loaded history. Taller bars represent price levels with high historical trading activity — used to identify support/resistance.

5.3 Analysis Log Panel (collapsible)

  • Visible only when an analysis log file exists for the symbol.
  • File discovery: the backend searches /app/investment_agent/analysis_log/ for files matching *{SYMBOL}*.html and *{SYMBOL}*.md. The most recently modified file is used.
  • HTML files: rendered inside a sandboxed <iframe> with an injected dark-theme CSS stylesheet. The panel is open by default for HTML files.
  • Markdown files: rendered with react-markdown + remark-gfm inside a styled scrollable container. The panel is collapsed by default for MD files.
  • The filename is shown above the content in small grey text.

5.4 Fibonacci Chart Panel (collapsible)

  • Visible only when a Fibonacci chart image file exists for the symbol.
  • File discovery: the backend searches /app/investment_agent/charts/Fibo/ for *{SYMBOL}*.png, *.jpg, *.jpeg, *.webp. The most recently modified file is used.
  • The image is returned as a Base64 data URI and displayed in a pan/zoom viewer.

5.5 Symbol Note Panel

  • Always visible at the bottom of the modal (not collapsible).
  • A resizable <textarea> pre-populated with any previously saved note for this symbol.
  • Save note button: calls PUT /analytics/note with the current text. On success, the button shows a green checkmark and "Saved" for 2.5 seconds.
  • Notes are per-user and per-symbol. The same symbol searched as SET and CRYPTO can have different notes (the asset_type field distinguishes them).
  • Empty / cleared notes: saving an empty textarea stores an empty string — it does not delete the note record.

6. File Naming Conventions for Knowledge Base Assets

For the Analysis Log and Fibonacci Chart panels to populate, files must exist in the investment agent directories with the symbol name somewhere in the filename.

Analysis logs — /app/investment_agent/analysis_log/

  • Accepted extensions: .html, .md
  • The symbol must appear in the filename (case-insensitive glob: *SYMBOL*.html)
  • Example: GULF_analysis_2026-05-30.html, BTC_weekly.md
  • Multiple files matching the same symbol: the most recently modified file is shown

Fibonacci charts — /app/investment_agent/charts/Fibo/

  • Accepted extensions: .png, .jpg, .jpeg, .webp
  • The symbol must appear in the filename
  • Example: GULF_fibo_2026-05.png, BTC_fibonacci.jpg
  • Multiple files: most recently modified wins
The symbol is validated before the filename glob runs. Symbols must match ^[A-Z0-9.\-]{1,20}$. Any symbol containing path characters (slashes, dots used as path separators, etc.) will be rejected with HTTP 400. Additionally, all glob results are verified to be physically inside the base directory — files outside the directory are silently dropped even if the glob matches them.

7. User Flows

7.1 Search and view chart

  1. Select asset type tab (SET / CRYPTO / DR)
  2. Type symbol in the search box → press Enter or click Search
  3. Symbol is added to the history table
  4. Analytics Modal opens automatically on found symbols
  5. Default chart interval is 1D
  6. Switch interval with the tab buttons — chart reloads
  7. Use dataZoom slider to focus on a time range

7.2 View analysis log for a symbol

  1. After modal opens, scroll down past the chart
  2. If an analysis log exists, the "Analysis Log" panel appears
  3. HTML logs are displayed in a dark-themed sandboxed iframe
  4. MD logs are rendered as formatted markdown (collapsed by default)
  5. Click the section header to toggle open/close

7.3 Save a note for a symbol

  1. Open Analytics Modal for any symbol
  2. Type or edit text in the Note textarea at the bottom
  3. Click Save note
  4. Confirmation animation shows for 2.5 s
  5. Note persists across sessions — next time the modal is opened for this symbol, the note is pre-filled

7.4 Re-open a previous search from history

  1. Click the symbol name (monospace bold) in the history table, OR click "View →" on any found entry
  2. Analytics Modal opens without making a new search API call
  3. The cached SearchResult from the original search is used
