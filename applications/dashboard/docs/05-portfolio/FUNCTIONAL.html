Portfolio Tracker — Functional Specification

1. Overview

The portfolio page (/portfolio) is the primary tracking interface. It is organised into four tabs. The default tab (Portfolio Overview) reads trade data from the Investment tracking.xlsx Excel file or from PostgreSQL (when in DB mode) and enriches open positions with live prices from Yahoo Finance. It provides comprehensive performance analytics with drill-down capability to individual transactions.

The Excel file is the single source of truth for all portfolio data in Excel mode. The application never writes to it. When new trades are recorded in the Excel file, the user clicks Refresh to copy the file into the application's working storage and reload the data.

1a. Tab navigation

A tab strip runs below the page header. The selected tab persists only for the duration of the browser session (no localStorage persistence). Tabs:

2. Excel file structure

The source file is an Excel workbook with a single sheet named Sheet1. Each row represents one trade entry (either open or closed). Column names must match exactly (including capitalisation).

ColumnTypeDescription
SymboltextThai SET ticker (e.g. GULF, PTT, KBANK)
DirectiontextLong or Short (case-insensitive)
Entry DatedateTrade entry date
Entry PricenumberPrice paid per share at entry
Exit DatedateExit date — blank for open positions
Exit Pricenumber or formulaExit price; blank or the string "NOT SELL" means position is still open
Position SizenumberNumber of shares
SLnumberStop-loss price (reference only — not enforced)
TPnumberTake-profit price (reference only)
RemarkstextOptional free-text notes for the trade
An open position is detected when Exit Price is NaN (blank cell) or the string "NOT SELL". Any other non-numeric value in this column may cause the position to be treated as closed with an incorrect exit price. Ensure the cell is either blank or contains a numeric value for closed trades.

6. Performance history chart

A dual-axis ECharts chart showing portfolio performance over time. The period selector (Daily / Weekly / Monthly) controls the granularity of both the chart and the Performance by Date table below it.

  • Left Y-axis: Cumulative P&L as a green area line chart (total accumulated profit/loss from the start of the filter period)
  • Right Y-axis: Period P&L as bar chart (profit or loss in each individual period) — green bars for positive periods, red for negative
  • X-axis: Dates grouped by the selected period
  • Tooltip: Shows both cumulative P&L and period P&L for the hovered date

The chart covers the date range defined by the From / To date filters.

7. Performance by date table

A collapsible table below the chart, grouped by the selected period (Daily / Weekly / Monthly). Each row represents one period bucket.

ColumnDescription
PeriodDate or date range label for the bucket
Net P&LTotal P&L for this period
Accumulated P&LRunning total from the start of the filter range
WinsCount of profitable trades in this period
LossesCount of losing trades
TotalTotal trades in this period
Win RatePercentage of trades that were profitable

Drill-down: Clicking any row opens a Transaction Modal showing every individual trade that occurred in that period. The modal columns are: Symbol, Direction, Entry Date, Exit Date, Entry Price, Exit Price, Size, Net P&L, P&L%, Remarks.

8. Performance by stock

A separate section below the date table showing P&L aggregated per stock symbol.

  • Bar chart: Horizontal bars sorted by P&L. Each bar is coloured green (profitable) or red (losing). Helps identify which stocks contributed most to overall performance.
  • Collapsible table: One row per symbol with columns: Symbol, Total Investment (cost basis), Current Value, Net P&L, P&L%, Wins, Losses, Win Rate (with a mini horizontal progress bar). Useful for identifying the most and least reliable stocks in the trading history.

9. Refresh

The Refresh button (top-right corner, circular arrow icon) triggers a multi-step process to reload the portfolio data from the source Excel file. A progress modal shows each step as it completes:

  1. Reading source file — shows the configured source path and file size
  2. Copying to working location — copies the Excel to /app/uploads/
  3. Clearing cache — writes the .cache_bust sentinel file, invalidating all 4 Uvicorn workers
  4. Reloading portfolio data — triggers TanStack Query refetch for all portfolio queries

The source path and working path are configured in Settings → App Configuration. If the source file is not found, the refresh modal shows a 503 error with the configured path displayed for troubleshooting.

The Objective tab appears between Portfolio Overview and Investment in the tab bar. It surfaces all portfolio_positions_db rows in a review-list layout and lets the user annotate each position with a decision rationale and a sentiment rating — capturing the why and how it felt behind every trade entry and exit.

10.3 Sort order

Positions are sorted by entry_date DESC (most recent first). Within the same entry date, open positions (null exit_date) appear before closed ones.

10.4 Inline editing — Reason

Each row has an editable textarea for the reason field. Typing and then clicking away (blur event) triggers an auto-save to the backend (PATCH /api/v1/objective/{id}). The field accepts plain text up to 5,000 characters. Clearing the field and blurring saves null.

10.5 Inline editing — Feel

Each row has a five-button picker using the same icon set as the Weekly Review List:

Clicking an icon saves immediately. Clicking the already-selected icon clears it (saves null).

Excel sync note: When the portfolio is in Excel mode and the user clicks Refresh, the sync process restores reason and feel values using a natural key (symbol, entry_date, position_size). If two positions share the same natural key (same stock, same date, same size), only one set of annotations will survive the sync. This is a known limitation.

11. Raw data viewer

The Raw Data button (table icon, top-right of the Portfolio Overview tab) opens a full-screen modal showing every row from the working-copy Excel file exactly as it was read by pandas. This is useful for debugging data issues — for example, if a position shows an unexpected status or P&L, the raw data view shows the exact values in each column before any processing.

The raw data table preserves the original Excel column names and includes all rows (including closed positions outside the current filter range).

A second button (Database icon) opens a Raw Source Data modal that reads the Excel file directly from the mounted source path (not the working copy). This is useful for checking whether the source volume is up to date when the working copy and source appear to differ.

Both Raw Data and Raw Source Data buttons are only shown when the active tab is Portfolio Overview.

12. DB Manager tab

The DB Manager tab provides access to the Portfolio DB CRUD manager directly from the portfolio page, without navigating to Settings. This is the primary CRUD interface when working in DB mode.

12.1 Mode gate

When the DB Manager tab is first activated, the app fetches the current portfolio mode. The displayed content depends on the mode:

  • Mode = db: The full position manager is shown (add, edit, sell, undo-sell, delete). This is identical to the interface in Settings → Portfolio DB.
  • Mode = excel: A placeholder is shown with a Database icon and the message "Portfolio is in Excel mode — switch to DB mode in Settings → Configuration to use the Portfolio Manager." No CRUD controls are shown.

12.2 Switching mode

The mode toggle (Excel ↔ DB) is not available on this tab. To switch mode, the user must navigate to Settings → Portfolio DB. After switching, returning to the DB Manager tab and refreshing shows the full CRUD manager.

12.3 CRUD operations

When in DB mode, the DB Manager tab exposes the full position lifecycle. For the complete functional description of each operation, refer to the Portfolio DB Mode spec, sections 3.3 – 3.8.

12.4 Status filter

A three-button filter bar at the top of the manager limits which positions are shown: Active, Closed, or All. The filter applies to the list fetch only — it does not affect total P&L shown in the summary line.

12.5 Partial-sell row layout

When a position has been partially sold, the sold portion appears as a child row immediately below its parent in the table. Child rows are visually indented with a corner-arrow indicator and labelled "sold". When the status filter hides the parent (e.g., parent is closed, filter is set to Active), the child row is still shown at the bottom of the table as an orphaned row.
