Dashboard — Functional Specification

1. Overview

The dashboard (/dashboard) is the landing page after login. It presents a high-level, at-a-glance summary of the portfolio status and market conditions. No user interaction is required — the page loads and auto-refreshes all data every 60 seconds.

The dashboard is designed to answer three questions at a glance: How is the portfolio performing right now? What is the capital allocation? What is happening in the broader market?

2. Widgets

2.1 Portfolio summary metrics

Four animated metric cards displayed in a responsive grid (2×2 on mobile, 1×4 on desktop). Each card shows a headline value with a sub-label for additional context. Values animate with a slide-up transition when data refreshes, making changes visually apparent.

2.2 Portfolio allocation chart

A donut chart showing the percentage of total portfolio value allocated to each open stock position. Each slice represents one stock, sized proportionally to its current market value (currentPrice × positionSize).

  • Labels show the stock ticker symbol
  • Tooltip shows both the percentage and the absolute THB value on hover
  • Stocks with very small allocations (<1%) are shown without labels to avoid clutter
  • If only one position is open, the donut shows a single full ring

Data source: GET /api/v1/portfolio-tracker/positions?status=active

2.3 Performance chart (mini)

A compact ECharts line chart showing cumulative P&L over the last 3 months with daily grouping. This is the same data used by the full performance chart on the Portfolio page, presented in a smaller format for the dashboard overview.

  • Y-axis shows cumulative P&L in THB
  • Green area fill below the line when P&L is positive
  • No interactive controls — it is a read-only summary view

2.4 Holdings table

A compact sortable table of open positions sorted by Net P&L (descending). Shows the top positions at a glance without requiring navigation to the full Portfolio page.

2.5 Market index bar

A horizontally scrollable strip across the top of the dashboard showing live market data. Updated on page load and every 60 seconds. Each index shows its name, current price, and daily change — coloured green for gains and red for losses.

Thai SET indices: SET, SET50, SET100, MAI, sSET — fetched from GET /api/v1/portfolio-tracker/market/set-indices

Global indices: S&P 500, NASDAQ, DOW, BTC-USD, Gold (GC=F) — fetched from GET /api/v1/portfolio-tracker/market/global-indices

Both endpoints call Yahoo Finance's quote API directly. If Yahoo Finance is temporarily unavailable, the index bar will show the last cached values or a loading state.

2.6 AI Insights widget

A small widget providing a shortcut to the AI Copilot chat. Displays a "Start chatting" prompt with suggested questions (e.g., "Summarise my portfolio", "What is my biggest winner?"). Clicking opens the /ai-copilot page.

3. Data refresh

All dashboard queries are managed by TanStack Query with consistent refresh settings:

  • staleTime: 60_000 — data is considered fresh for 60 seconds after fetching; no refetch during this window
  • refetchInterval: 60_000 — auto-polls every 60 seconds when the tab is in focus
  • Refetch on window focus is enabled by default — switching back to the browser tab will trigger a refresh if data is stale

The market index strip and the portfolio summary widgets both refresh independently on their own 60-second cycles.

4. Empty state

If no portfolio data is available — for example, when the Excel source file has not been configured or the configured path does not exist — all portfolio widgets display a placeholder message:

"No portfolio data available. Go to Settings → App Configuration to set the Excel source file path."

The market index bar continues to show live data even when portfolio data is unavailable, since it fetches independently from Yahoo Finance and does not depend on the Excel file.

5. Chart library — ECharts

All charts on the dashboard use the echarts-for-react wrapper around Apache ECharts 5. Charts are rendered via the ReactECharts component with animation: false for snappier rendering during data refresh cycles.

The dark theme is applied through ECharts options rather than a theme file:

  • backgroundColor: '#0d1117' — matches the page background
  • textStyle.color: '#64748b' — axis labels and tooltips use the muted grey
  • Positive values: #3fb950 (green), Negative values: #f85149 (red)

The shared EChartsChart component (frontend/src/components/analytics/EChartsChart.tsx) provides the full candlestick / OHLCV chart used in the Analytics modal and Weekly Scan evaluate view. See the Weekly Scan technical documentation for the complete chart feature set including VRVP, RSI, and Stochastic panels.
