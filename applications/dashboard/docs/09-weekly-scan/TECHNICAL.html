3. Alembic migrations

The Weekly Scan module introduced Alembic to the project. All four weekly scan tables are created in the initial Alembic migration. The env.py uses the async engine pattern for SQLAlchemy 2.0:

# backend/alembic/env.py
from app.models import *    # imports all models to register them with Base.metadata

# Run with:

create_all() at startup is still present for development convenience (new tables created without a migration will be added automatically), but Alembic is the canonical migration path for production.

4. Backend API endpoints

Router prefix: /api/v1/weekly-scan — all endpoints require Bearer token.

Scan creation logic

When POST /scans is called, the backend first checks for named symbol lists. If none exist, it falls back to the legacy UserScanConfig or the built-in SET50 defaults.

# Priority 1: named symbol lists (current model)
lists = await db.execute(
    select(UserSymbolList)
    .where(UserSymbolList.user_id == uid)
    .order_by(UserSymbolList.sort_order)
)
if symbol_lists := lists.scalars().all():
    for sl in symbol_lists:
        for i, sym in enumerate(sl.symbols):
            db.add(WeeklyScanItem(
                scan_id=scan.id, symbol=sym,
                list_name=sl.name, market=sl.market, sort_order=i
            ))
else:
    # Priority 2: legacy UserScanConfig or SET50_DEFAULT
    config = await db.scalar(select(UserScanConfig).where(UserScanConfig.user_id == uid))
    symbols = config.symbols if config else SET50_DEFAULT
    for i, sym in enumerate(symbols):
        db.add(WeeklyScanItem(scan_id=scan.id, symbol=sym, sort_order=i))

Partial item update

The PUT .../items/{symbol} endpoint uses Pydantic's model_dump(exclude_unset=True) to write only the fields that were explicitly included in the request body. This allows a single colour change without affecting buy price, TP, SL, etc.

@router.put("/scans/{scan_id}/items/{symbol}")
async def update_item(scan_id, symbol, body: ItemEval, _: UserId, db):
    item = await _get_item_or_404(scan_id, symbol, db)
    for field, val in body.model_dump(exclude_unset=True).items():
        setattr(item, field, val)
    # Explicitly bump scan updated_at — SQLAlchemy onupdate does not
    # fire when only child rows change
    await db.execute(
        text("UPDATE weekly_scans SET updated_at = now() WHERE id = :id"),
        {"id": scan_id}
    )
    await db.commit()
    return item

5. Week prices — concurrency model

Week prices are fetched for all symbols in a scan concurrently using asyncio.gather with a semaphore to cap concurrent Yahoo Finance connections at 5.

@router.get("/scans/{scan_id}/week-prices")
async def get_week_prices(scan_id: uuid.UUID, _: UserId, db: AsyncSession):
    scan = await _get_scan_or_404(scan_id, db)
    monday, friday = _parse_week_dates(scan.name)   # from WEEKLY_SCAN_DD_MM_YYYY
    symbols = [item.symbol for item in scan.items]
    sym_market = {item.symbol: item.market for item in scan.items}

    loop = asyncio.get_running_loop()
    sem = asyncio.Semaphore(5)    # max 5 concurrent Yahoo Finance requests

    async def _fetch(sym: str) -> tuple[str, dict]:
        async with sem:
            result = await loop.run_in_executor(
                None,
                _fetch_sym_prices,      # synchronous yfinance call
                sym, monday, friday, sym_market.get(sym, 'SET')
            )
        return sym, result

    pairs = await asyncio.gather(*[_fetch(s) for s in symbols])
    return {"mon_date": str(monday), "fri_date": str(friday), "prices": dict(pairs)}

Ticker symbol translation

def _sym_to_ticker(symbol: str, market: str = 'SET') -> str:
    sym = symbol.strip().upper()
    # DR symbols and crypto (e.g. BTCUSD-DR, GC=F) — use as-is
    if market == 'CRYPTO' or sym.endswith('-DR') or '=' in sym:
        return sym
    # HK: zero-pad symbol to 4 digits + .HK suffix
    if market == 'HK':
        return f"{sym.zfill(4)}.HK"
    # US and OTHER: bare symbol, no suffix
    if market in ('US', 'OTHER'):
        return sym
    # SET default: append .BK
    return f"{sym}.BK"

Inline editing micro-components

Three micro-components handle inline editing across all table cells:

  • NumCell — click to enter edit mode with <input type="number">; blur or Enter commits; Escape cancels; empty string saves as null
  • TextCell — same pattern for free-text fields (remark, strategy free-text)
  • ColorPicker — five coloured dot buttons; clicking an active dot deselects; clicking inactive activates

Each change calls updateField(symbol, {field: value}) which issues PUT .../items/{symbol} and then invalidates the ['weekly-scan', id] query to refresh the table.

Portfolio auto-mark implementation

const markPortfolioSymbols = async () => {
  // Fetch live positions
  const positions = await portfolioTrackerService.getPositions({ status: 'active' })
  const activeSymbols = new Set(positions.map(p => p.symbol.toUpperCase()))

  // Build strategy lookup from latest purchase plan (optional)
  const allPlans = await actionPlanService.list('purchase', null)
  let strategyMap: Record<string, string> = {}
  if (allPlans.length > 0) {
    const latest = await actionPlanService.get(allPlans[0].id)
    for (const item of latest.purchase_items ?? []) {
      if (item.strategy) strategyMap[item.stock.toUpperCase()] = item.strategy
    }
  }

  // Mark matching scan items as PURPLE in parallel
  const toMark = scan.items.filter(i => activeSymbols.has(i.symbol.toUpperCase()))
  await Promise.all(toMark.map(item =>
    weeklyScanService.upsertItem(id, item.symbol, {
      color_mark: 'PURPLE',
      strategy: strategyMap[item.symbol.toUpperCase()] ?? item.strategy ?? undefined,
    })
  ))
  await queryClient.invalidateQueries({ queryKey: ['weekly-scan', id] })
}

7. Evaluate view — queue building

function buildQueue(scan: WeeklyScan, mode: string, list: string | null): WeeklyScanItem[] {
  let items = [...scan.items].sort((a, b) => a.symbol.localeCompare(b.symbol))

  // Filter by list tab if not "All"
  if (list) items = items.filter(i => i.list_name === list)

  // Filter by mode
  if (mode === 'remaining') return items.filter(i => !i.color_mark)
  if (mode.startsWith('color_')) {
    const color = mode.slice(6) as ColorMark
    return items.filter(i => i.color_mark === color)
  }
  return items   // 'all' mode
}

Mode and list are passed as URL search parameters (?mode=remaining&list=SET50), allowing direct URL linking into a specific evaluation queue.

DataZoom implementation

The start position of the zoom window is computed so the visible area covers exactly the last N calendar days, regardless of how many candles are in the data (varies by interval and trading days):

function zoomStartForRange(dates: string[], days: number): number {
  if (!dates.length || days >= 912) return 0
  const cutoff = new Date()
  cutoff.setDate(cutoff.getDate() - days)
  const cutoffStr = cutoff.toISOString().slice(0, 10)
  const idx = dates.findIndex(d => d.slice(0, 10) >= cutoffStr)
  if (idx <= 0) return 0
  return (idx / dates.length) * 100   // percentage of the total x-axis
}

VRVP (Volume at Price)

Horizontal bar overlay on the price panel showing volume distribution across price levels. Bars are normalised to a maximum of 15% of the chart width:

// Build VRVP histogram
const priceMin = Math.min(...lows)
const priceMax = Math.max(...highs)
const bucketSize = (priceMax - priceMin) / NUM_VRVP_BUCKETS    // e.g. 50 buckets
const volumeByBucket = new Array(NUM_VRVP_BUCKETS).fill(0)

for (let i = 0; i < closes.length; i++) {
  const bucket = Math.floor((closes[i] - priceMin) / bucketSize)
  volumeByBucket[Math.min(bucket, NUM_VRVP_BUCKETS - 1)] += volumes[i]
}

const maxVol = Math.max(...volumeByBucket)
const vrvpSeries = volumeByBucket.map((vol, i) => ({
  value: [vol / maxVol * 15,    // width as % of chart (max 15%)
          priceMin + i * bucketSize,
          priceMin + (i + 1) * bucketSize],
}))

Custom tooltip

A single cross-hair tooltip shows OHLCV values for the candlestick, volume, RSI, and Stochastic %K/%D at the current cursor position. All four panels share the same axisPointer for synchronised highlighting.

11. Model registration

All weekly scan models must be imported in backend/main.py before Base.metadata.create_all:

from app.models.weekly_scan import (       # noqa: F401
    UserScanConfig, UserSymbolList, WeeklyScan, WeeklyScanItem
)
Forgetting to import a model here will result in the corresponding table not being created by create_all(), and Alembic migrations will also fail to detect the table. Always add new model classes to this import block.
