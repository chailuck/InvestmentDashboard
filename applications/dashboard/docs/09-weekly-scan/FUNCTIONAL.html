Weekly Scan — Functional Specification

1. Overview

The Weekly Scan module (/weekly-scan) supports a disciplined, repeatable workflow for reviewing a watchlist of SET and global stocks once per week. The analyst creates a named scan (typically dated to the upcoming Saturday), works through each symbol to evaluate its chart setup using the built-in OHLCV candlestick chart, assigns a colour mark and trading strategy, and optionally promotes selected symbols directly to a purchase action plan.

The module is personal — each user manages their own scans, symbol lists, and configuration independently. One user's scans are not visible to other users.

2. Core concepts

2.1 Symbol lists

A user maintains one or more named symbol lists (e.g., "SET50", "Crypto DRs", "US Watchlist"). Each list is associated with a market context which determines how Yahoo Finance ticker symbols are constructed. When a scan is created, symbols are drawn from all lists and grouped by list name inside the scan.

  • Lists are ordered by sort_order; symbols within a list are stored in the user-defined order
  • New users who have no lists configured will have a "Default" list automatically seeded with SET50 constituent tickers plus two DR symbols (BTCUSD-DR, GOLUSD-DR) on first access

2.2 Scans

A scan is a dated snapshot of symbols from all active symbol lists. Its name follows the convention WEEKLY_SCAN_DD_MM_YYYY where the date is the target Saturday. The system auto-suggests the next Saturday's name if the user already has prior scans.

Once created, a scan is a fixed snapshot — it does not automatically update when the underlying symbol lists change. The user must click Refresh to merge new symbols from current lists into an existing scan. Existing items with evaluation data are never overwritten by a refresh.

2.3 Colour mark system

Each item in a scan receives exactly one colour mark as the primary evaluation verdict. The marks form a priority queue from top trading candidates (CYAN) down to portfolio positions (PURPLE).

2.5 Trade parameters

Each item can store optional trade parameters alongside the colour and strategy. These are editable inline in the table or in the evaluate view:

  • Buy Price — intended entry price
  • Size — number of shares
  • TP — take-profit target price
  • SL — stop-loss price
  • Remark — free-text note (e.g., "Breakout from 3-month base", "Watch earnings date")

3. User workflows

3.1 Managing symbol lists

  1. Navigate to Weekly Scan and click the Symbol Lists settings button
  2. Create one or more named lists and assign a market context: SET, US, HK, CRYPTO, or OTHER
  3. Enter the list of symbols (one per line, or comma-separated); the market context determines how tickers are passed to Yahoo Finance
  4. Reorder lists by setting a sort order; the order affects the tab layout in the scan detail view

3.2 Creating a scan

  1. Navigate to Weekly Scan in the sidebar
  2. Click New Scan. The suggested name is pre-filled (e.g., WEEKLY_SCAN_07_06_2026)
  3. Confirm. The backend creates the scan and populates it with all symbols from all active symbol lists, preserving list grouping
  4. The browser navigates to the new scan's detail page

3.3.1 Previous Scan column

The scan detail table includes a read-only Prev column that shows how each symbol was evaluated in the immediately preceding scan — the scan created most recently before the current one, by timestamp, belonging to the same user.

The column header displays "Prev" with a sub-label showing the date portion of the previous scan's name (the WEEKLY_SCAN_ prefix is stripped, e.g. 22_06_2025). When no predecessor scan exists the column header sub-label is blank and all cells show —.

Cell display logic:

ConditionCell content
Symbol is not present in the previous scan—
Symbol is present but unevaluated (no colour, no strategy)Hollow dot + —
Colour mark set, strategy is nullFilled dot in the previous colour + —
Both colour mark and strategy are setFilled dot in the previous colour + strategy short code (e.g. BOD, BO, ททกต)

The strategy abbreviation is the same short code shown in the Strategy column (see section 2.4). For custom (OTHERS) strategies, the first 6 characters are shown with the full text available in a tooltip on hover.

This column is read-only. It cannot be edited from the scan table or the Evaluate view. It reflects the saved state of the previous scan at the time the page loads and does not update as the current scan is being evaluated.

Filter controls above the table:

  • List tabs — one tab per symbol list; "All" shows every symbol
  • Symbol filter — text search on ticker symbol
  • Color filter — multi-select colour badge buttons
  • Strategy filter — text search on strategy name

3.4 Evaluate view (full-screen chart)

Click Start Scan to enter the full-screen Evaluate view (/weekly-scan/[id]/evaluate). Choose a mode and optionally a specific list before starting.

Available modes:

  • All symbols — every symbol in the selected list
  • Remaining only — symbols with no colour mark yet (unreviewed)
  • CYAN / GREEN / YELLOW / etc. — symbols already marked with a specific colour

Layout:

  • Left panel: Scrollable symbol queue showing all symbols in the queue with their current marks
  • Centre: EChartsChart component — OHLCV candlestick with volume, RSI, and Stochastic panels; interval switcher (1h / 4h / 1d / 1wk)
  • Right panel: Evaluation form — colour picker, strategy selector, buy/size/TP/SL/remark inputs
  • Top bar: Live price with day change % fetched from the analytics service

Navigation: Next and Prev buttons advance through the queue. Selecting RED automatically skips to Next. Save & Close saves the current evaluation and returns to the scan table.

3.5 Scan dashboard

The Scan Dashboard (/weekly-scan/[id]/dashboard) provides a summary view of a completed scan — useful for planning what to do with the week's findings:

  • Stat cards: Total symbols, evaluated count, CYAN+GREEN count (actionable), best/worst week performer
  • Colour distribution donut chart: Shows the breakdown of marks (Pending shown as grey)
  • Bar charts: Week % return grouped by colour mark and by strategy
  • Filtered symbol tables: Symbols grouped by colour mark for quick reference

3.6 Adding symbols to a purchase plan

From the scan table, click the cart icon on any row to open the Add to Plan modal. The modal lists the five most recent purchase plans. Select a plan and confirm to append the symbol with all its evaluation data (buy price, size, TP, SL, strategy) as a new row in that purchase plan. The purchase plan editor then opens automatically so the user can review and save.

3.7 Mark Portfolio Symbols

The Mark Portfolio button in the scan toolbar cross-references all active portfolio positions against the scan's symbol list. Matching symbols are automatically marked as PURPLE. If the latest purchase plan contains a strategy for the same symbol, that strategy is also inherited — avoiding manual re-entry for known positions.

4. Week price calculation

Monday open and Friday close prices are fetched from Yahoo Finance using the date embedded in the scan name (WEEKLY_SCAN_DD_MM_YYYY). Ticker translation by market:

MarketSuffix ruleExample
SET{SYMBOL}.BKGULF → GULF.BK
US / OTHERNo suffixAAPL → AAPL
HKZero-pad to 4 digits + .HK700 → 0700.HK
CRYPTO / DR symbolsNo suffixBTCUSD-DR → BTCUSD-DR

If the scan week is in the future (Monday has not yet occurred), the most recent close is used as a proxy for Monday open. All symbols are fetched concurrently with a semaphore of 5 to avoid rate-limiting.

5. Sidebar widget — clickable symbols

The Weekly Scan sidebar widget shows the most recent scan's CYAN (Potential) and GREEN (Good) symbols as clickable links alongside the colour distribution dots. This allows investors to jump straight into a stock's analytics without navigating away from their current page.

5.1 Interaction

  • Each CYAN and GREEN symbol is rendered as a button in the sidebar widget
  • Clicking any symbol opens the Analytics modal scoped to that symbol, presenting the OHLCV chart, indicators, and live price data
  • Symbols are separated by a · centred dot; no trailing separator is rendered after the last symbol in each group
  • Only one modal can be open at a time — opening a second symbol closes the first automatically
  • Closing the modal (via the modal's close button or Escape) returns the sidebar to its unchanged state
  • If a list is empty (no CYAN or no GREEN symbols in the latest scan), that group is hidden entirely and no buttons are rendered

5.2 Keyboard accessibility

  • Symbols are native <button> elements, so they participate in the Tab order automatically
  • Pressing Enter or Space on a focused symbol button opens the analytics modal
  • Each button carries a title attribute ("Open analytics for {SYMBOL}") for screen reader and tooltip support
