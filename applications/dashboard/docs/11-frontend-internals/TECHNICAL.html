This document covers the TypeScript type system, service layer, Axios client configuration, Zustand state management, and WebSocket integration. A developer can rebuild the entire frontend layer from this reference without reading source files.

2. Axios API Client

Source: frontend/src/services/api.ts

2.1 Configuration

import axios from 'axios'

// All requests go through the Next.js server-side proxy.
// This avoids CORS and keeps the backend URL off the client entirely.
const BASE_URL = '/api/proxy'

export const apiClient = axios.create({
  baseURL: `${BASE_URL}/api/v1`,
  headers: { 'Content-Type': 'application/json' },
  timeout: 30_000,          // 30 s hard timeout
})
The Next.js proxy route lives at src/app/api/proxy/[...path]/route.ts. It forwards every request to the backend's internal Docker hostname (http://backend:8000), adding no extra headers.

2.2 Request Interceptor — Attach Bearer Token

apiClient.interceptors.request.use((config) => {
  const token = useAuthStore.getState().accessToken
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

2.3 Response Interceptor — Auto-Refresh on 401

apiClient.interceptors.response.use(
  (res) => res,
  async (error) => {
    const original = error.config
    if (error.response?.status === 401 && !original._retry) {
      original._retry = true          // prevent infinite loop
      try {
        const { refreshToken, setTokens } = useAuthStore.getState()
        const { data } = await axios.post('/api/proxy/api/v1/auth/refresh', { refreshToken })
        setTokens(data)               // update Zustand store
        original.headers.Authorization = `Bearer ${data.accessToken}`
        return apiClient(original)    // retry original request
      } catch {
        useAuthStore.getState().clearAuth()   // logout on refresh failure
      }
    }
    return Promise.reject(error)
  }
)

2.4 Error Extraction Helper

export function extractApiError(err: unknown): string {
  if (axios.isAxiosError(err)) {
    const data = err.response?.data
    return data?.detail ?? data?.message ?? err.message ?? 'Request failed'
  }
  return 'An unexpected error occurred'
}

3. State Management

The frontend uses Zustand for global client state and TanStack Query (React Query) for server data. They serve distinct roles.

Store / LayerWhat it ownsWhen to use
Zustand authaccessToken, refreshToken, current User, isAuthenticatedAny component that needs auth state or token manipulation
Zustand dashboardwidget layout (WidgetConfig[]), sidebar open/closedPersisted UI state that survives page navigation
Zustand notificationsNotification[], unread countToast queue and notification bell
TanStack QueryAll server data (positions, indices, action plans, scan items)Any data fetched from the API — handles caching, refetch, loading/error states
React useStateLocal ephemeral UI state (modal open, form values)Component-local state with no sharing requirement

4. WebSocket / Socket.IO Integration

Source files: frontend/src/websocket/client.ts, frontend/src/websocket/hooks.ts

4.1 WebSocketClient Class

// Singleton exported as wsClient
export const wsClient = new WebSocketClient()

class WebSocketClient {
  // Connection
  connect(): void       // idempotent — no-op if already connected
  disconnect(): void    // disconnects and nulls the socket

  // Room subscriptions
  subscribe(portfolioId: string): void   // emits 'subscribe' with channel 'portfolio:{id}'
  unsubscribe(portfolioId: string): void // emits 'unsubscribe'

  // Event handlers (returns unsubscribe fn)
  on<T>(event: WSEventType, handler: (payload: T) => void): () => void

  // Accessors
  get isConnected(): boolean
  get id(): string | undefined
}

4.2 Connection Configuration

// WS_URL from env (default: ws://localhost:8000)
const WS_URL = process.env.NEXT_PUBLIC_WS_URL ?? 'ws://localhost:8000'

io(WS_URL, {
  auth: { token: accessToken },       // JWT sent in Socket.IO handshake auth
  transports: ['websocket'],          // skip long-polling upgrade
  reconnection: true,
  reconnectionAttempts: 5,
  reconnectionDelay: 1_000,           // 1 s initial
  reconnectionDelayMax: 10_000,       // 10 s max
})
The backend's Socket.IO connect handler logs the connection but does NOT currently validate the JWT. The auth.token is available in the auth dict but the TODO has not been implemented. Treat the WS connection as unauthenticated at the transport level until that TODO is resolved.

4.3 React Hooks

useWebSocket — global connection lifecycle

const { isConnected, wsClient } = useWebSocket()
// Calls wsClient.connect() on mount.
// Polls wsClient.isConnected every 2 s to keep state in sync.
// Does NOT disconnect on unmount (connection is shared).

useWSEvent — subscribe to a single event type

useWSEvent<QuotePayload>('quote_update', (payload) => {
  // handler is stable via useRef — no re-subscription on re-render
  console.log(payload.symbol, payload.price)
})
// Unsubscribes automatically on component unmount.

useRealtimeQuote — live price with flash animation

const { quote, flash } = useRealtimeQuote('PTT')
// quote: { price, change, changePct } | null
// flash: 'up' | 'down' | null  (cleared after 400 ms)
// Only updates if payload.symbol matches the requested symbol.

6. THB Number Formatting

Thai Baht values are formatted with compact notation for large numbers. Use this helper pattern in any component that displays P&L or monetary values:

function fmtTHB(n: number): string {
  const sign = n >= 0 ? '+' : ''
  if (Math.abs(n) >= 1_000_000) return `${sign}${(n / 1_000_000).toFixed(2)}M ฿`
  if (Math.abs(n) >= 1_000)     return `${sign}${(n / 1_000).toFixed(1)}K ฿`
  return `${sign}${n.toLocaleString('th-TH', { minimumFractionDigits: 0, maximumFractionDigits: 0 })} ฿`
}
// Examples: +1.23M ฿  |  -45.6K ฿  |  +3,200 ฿
