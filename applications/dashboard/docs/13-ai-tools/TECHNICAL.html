AI Copilot — Tool Reference & Technical Design

This document covers the AI copilot's tool definitions, execution logic, SSE streaming format, session management, and system prompt construction. Source: backend/app/api/v1/endpoints/ai.py.

1. Architecture Overview

AspectDetail
LLM providerAnthropic API — client.messages.create()
ModelConfigurable via AI_DEFAULT_MODEL env var (default: claude-sonnet-4-6)
API key env varANTHROPIC_API_KEY — must be set or the copilot returns an error message
StreamingServer-Sent Events (SSE) — text/event-stream response. The Anthropic SDK call is synchronous and run in a thread pool via asyncio.to_thread(); text is then chunked word-by-word into the SSE stream.
Max loop iterations6 — prevents runaway tool-call chains
Session storageIn-process Python dict _SESSIONS — not shared across workers, not persisted on restart
Session history limitLast 20 messages retained per session
Agent directory/app/investment_agent (Docker volume mount)
Sessions are stored in-memory per worker. If Uvicorn runs multiple workers, a user's conversation may hit different workers on different requests, resetting context. For multi-worker deployments, session state must be externalised to Redis.

3. SSE Stream Event Format

Each event is a line of the form data: {JSON}\n\n. The frontend reads this via EventSource or a streaming fetch and processes each data: payload.

Event typeAdditional fieldsWhen emitted
textcontent: stringEach word of LLM response text, space-separated
tool_startname: stringImmediately before a tool is called
tool_endname: stringImmediately after a tool returns
errorcontent: stringAnthropic API error or missing API key; stream ends after this
donesession_id: stringStream complete — final event, always present
# Example event stream for "/portList"
data: {"type": "tool_start", "name": "get_portfolio_positions"}

data: {"type": "tool_end", "name": "get_portfolio_positions"}

data: {"type": "text", "content": "Here "}
data: {"type": "text", "content": "are "}
data: {"type": "text", "content": "your "}
data: {"type": "text", "content": "active "}
data: {"type": "text", "content": "positions:\n\n"}
...
data: {"type": "done", "session_id": "3f9a1c2e-..."}

4. System Prompt Construction

The system prompt is built dynamically on each request by _system_prompt(). Parts are concatenated with \n\n---\n separators.

  1. Agent identity — reads /app/investment_agent/CLAUDE.md if it exists; otherwise uses a hardcoded fallback: "You are a personal stock investment assistant for Thai SET stocks and DRs…"
  2. Personal trading style — appends /app/investment_agent/knowledge/personal-trading-style.md if it exists (section header: # Personal Trading Style)
  3. Dashboard instructions — hardcoded section that maps slash-commands to tools:
    • /portList → call get_portfolio_positions
    • /portAction, /portPlan → call get_portfolio_positions + get_performance_summary
    • /portHist → call get_performance_summary
    • /analyze TICKER → call get_live_price then read_knowledge_doc
To customise the copilot's personality or add new slash-command mappings, edit /app/investment_agent/CLAUDE.md (mounted volume). Changes take effect on the next request — no restart needed.

5.2 get_live_price

Input parameterTypeRequiredDescription
tickerstringYesThai SET symbol, DR code, or fully qualified Yahoo ticker

Ticker normalisation:

ticker = ticker.strip().upper()
is_dr  = bool(re.match(r"^[A-Z]+\d+$", ticker))   # e.g. AAPL01, NVDA06
is_bk  = ticker.endswith(".BK")
yf_ticker = ticker if (is_dr or is_bk) else ticker + ".BK"

Return format:

**PTT** 📈
- Price: **37.50 THB**
- Change: +2.50 (+7.14%)
- Source: Yahoo Finance (SET data may be 15 min delayed)

Error return: "Could not fetch price for **{ticker}** ({yf_ticker}). Market may be closed or ticker not found on Yahoo Finance."

Execution: Calls both get_daily_performance(from_date, period=period) and get_performance_by_stock(from_date). Only closed (realized) trades are included in the period buckets. The by-stock view includes open positions.

5.4 read_knowledge_doc

Input parameterTypeRequiredDescription
filenamestringYesFilename within /app/investment_agent/knowledge/, e.g. knowledge_fibo.md

Path resolution: /app/investment_agent/knowledge/{filename}

Return: First 6000 characters of the file content (UTF-8, encoding errors replaced).

Error return (file not found):

"Document 'foo.md' not found. Available: knowledge_fibo.md, technical-analysis.md, personal-trading-style.md"
Knowledge documents must be .md files placed in the knowledge/ subdirectory of the agent directory. The tool uses glob("*.md") to list available files — subdirectories are not searched.

6. Tool Execution Loop

for iteration in range(6):                           # max 6 rounds
    response = client.messages.create(               # synchronous call in thread
        model=settings.ai_default_model,
        max_tokens=4096,
        system=system_prompt,
        messages=session_history,
        tools=TOOLS,
    )

    # Stream text chunks word-by-word
    for word in full_text.split(" "):
        yield SSE text event

    session_history.append(assistant_message)

    if response.stop_reason != "tool_use":
        break                                        # LLM finished, exit loop

    # Execute all requested tools in sequence
    for tool_use in tool_uses:
        yield SSE tool_start
        result = _execute_tool(tool.name, tool.input)
        yield SSE tool_end

    session_history.append(tool_results_message)
    # → next iteration: LLM receives tool results and continues

session_history = session_history[-20:]              # trim to last 20 messages
yield SSE done

7. Error Handling

Failure modeBehaviour
ANTHROPIC_API_KEY not set or placeholderImmediately yields { type: "error" } + { type: "done" } without calling the API
Anthropic APIErrorYields { type: "error", content: str(exc) } + { type: "done" } and exits
Tool raises any exception_execute_tool catches all exceptions and returns "Tool error [{name}]: {exc}" as a string — loop continues
Script timeout (60 s)Caught by the generic except Exception in _execute_tool — returns error string to LLM
Max iterations reachedLoop exits silently; final done event is emitted

8. Adding a New Tool

To expose a new capability to the AI copilot:

  1. Define the tool schema — add an entry to the TOOLS list in ai.py: 
    {
    "name": "my_new_tool",
    "description": "What this tool does and when the LLM should call it.",
    "input_schema": {
        "type": "object",
        "properties": {
            "param": { "type": "string", "description": "..." }
        },
        "required": ["param"],
    },
}
  2. Implement execution — add an elif name == "my_new_tool": branch inside _execute_tool(). Always return a plain string — the LLM reads this as a tool result.
  3. Update the system prompt — add a line to the Dashboard AI Instructions section so the LLM knows when to call the new tool.
  4. Document the tool — add an entry to this document (section 5) with input parameters and return format.
