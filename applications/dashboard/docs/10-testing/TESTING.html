Testing Strategy & Test Cases

1. Overview

The test suite covers the full InvestmentDashboard stack. Two automated suites run in parallel and are unified under a single shell script entry point.

# Backend only
bash applications/dashboard/run-tests.sh backend

# Frontend only
bash applications/dashboard/run-tests.sh frontend

2. Backend Test Suite

2.1 Prerequisites

The backend tests use a dedicated investment_test_db postgres database. Create it once:

docker exec inv_postgres psql -U postgres -c "CREATE DATABASE investment_test_db;"
Tests drop and recreate all tables at session start. Never point TEST_DATABASE_URL at the production database.

2.2 Configuration — backend/pytest.ini

[pytest]
asyncio_mode = auto
testpaths = tests
addopts = -v --tb=short --cov=app --cov-report=term-missing --cov-report=html:htmlcov
filterwarnings = ignore::DeprecationWarning

2.3 Fixtures — backend/tests/conftest.py

FixtureScopeDescription
enginesessionCreates async engine against investment_test_db. Runs Base.metadata.create_all at start and drop_all at end.
db_sessionfunctionOpens a session, yields it, rolls back after each test — full isolation.
clientfunctionAsyncClient with get_db overridden to use db_session. Base URL: http://test.
auth_clientfunctionCreates a test user directly in the DB (bypasses admin-only registration route), mints a JWT, returns an authenticated AsyncClient.
Important: There is no public POST /auth/register endpoint. User creation is admin-only (POST /api/v1/users). The auth_client fixture inserts the user via SQLAlchemy directly and mints a token using the app's JWT utility — it does not go through the HTTP layer for setup.
Important: The FastAPI app is wrapped in a Socket.IO middleware. Use fastapi_app (not app) when constructing ASGITransport. Import it with from main import fastapi_app.

# Single test

# Using the helper script

4. Master Test Runner

File: applications/dashboard/run-tests.sh

bash run-tests.sh            # backend + frontend
bash run-tests.sh backend    # backend only
bash run-tests.sh frontend   # frontend only

The script detects whether the inv_backend Docker container is running and executes pytest inside it. If not, it falls back to running pytest locally (requires backend venv with deps installed).

5. Adding Tests for New Features

Backend (new endpoint)

  1. Add a new async def test_*(auth_client) function in the appropriate tests/test_*.py file
  2. Use auth_client for authenticated calls; client for unauthenticated tests
  3. Assert response.status_code and key fields in response.json()
  4. For endpoints that call yfinance or read Excel, patch with monkeypatch.setattr()
  5. Run: docker exec inv_backend bash -c "cd /app && pytest tests/test_NEW.py -v"

Frontend (new widget)

  1. Create src/components/dashboard/__tests__/NewWidget.test.tsx
  2. Mock the service: vi.mock('@/services/portfolioTracker', () => ({ portfolioTrackerService: { getX: vi.fn().mockResolvedValue(...) } }))
  3. Import render from @/test/test-utils (provides QueryClient)
  4. Use waitFor(() => expect(screen.getByText('...')).toBeInTheDocument()) for async
  5. Run: npx vitest run src/components/dashboard/__tests__/NewWidget.test.tsx

6. Manual Acceptance Test Checklist

Run through all 12 scenarios after each significant deployment.

  1. Auth: Register → Login → View dashboard → Logout → Login again with same credentials
  2. Dashboard layout: Click "Edit layout" → Drag Market Pulse widget → Click "Lock layout" → Reload page → Widget stays in new position
  3. Market Pulse Widget: Verify SET50/SET100/sSET and S&P500/NASDAQ/DOW/BTC/Gold all display with change %
  4. Scan-to-plan flow: Create weekly scan → Evaluate 3 symbols → Mark 1 CYAN + 1 GREEN → Scan Dashboard → "Export CYAN+GREEN to Plan" → Select plan → Verify plan has 2 items, no duplicates
  5. Keyboard shortcuts: Open evaluate view → Press 1 → symbol marked CYAN → Press → → next symbol → Press S → saved successfully
  6. Symbol Notes: Open evaluate for ADVANC → Type note → Click away (blur) → "Saved" flash appears → Reopen same symbol → Note persists
  7. Sync from Portfolio: Settings → Symbol Lists → "Sync from Portfolio" → Count of new symbols matches active positions not already in list
  8. P&L Waterfall: Toggle 1M → bars change → Toggle 3M → bars change → Toggle All → shows lifetime data
  9. Scan Heat Tile: Ensure ≥1 CYAN/GREEN scan pick is also an active position → Widget shows colored tile for that symbol with correct P&L%
  10. Documents: Open all 10 sections → Each renders correctly in HTML iframe → Application Guide sidebar nav scrolls to correct section on click
  11. Bulk Export deduplication: Export CYAN+GREEN from two different scans to the same plan → No duplicate stock entries in plan
  12. Portfolio Tracker refresh: Click refresh in portfolio page → Data reloads → Performance chart updates
