AI Copilot — Functional Specification

1. Overview

The AI Copilot (/ai-copilot) is a streaming chat interface powered by Anthropic Claude. Unlike a generic chatbot, the Copilot has direct access to the user's portfolio data, live stock prices, and a personal trading knowledge base. Responses stream token-by-token for a real-time feel, and the AI can autonomously call tools to retrieve live data before answering.

The Copilot can answer complex questions like "Which of my open positions has the best risk/reward right now?" by fetching live prices, reading the Excel portfolio data, and reasoning over both in a single response.

The AI Copilot requires ANTHROPIC_API_KEY to be set in backend/.env. Without it, the endpoint returns a 400 error and the UI displays an informational error message.

2. Interface

  • Chat history: Scrollable list of alternating user and assistant messages, rendered as Markdown (code blocks, tables, bold/italic all render correctly)
  • Input box: Multi-line text input; Enter sends the message; Shift+Enter inserts a newline
  • Session controls: A New Session button clears the conversation history both on the backend (Redis/memory) and in the UI, and assigns a new session ID
  • Tool indicators: While the AI calls a tool, an inline chip appears showing which tool is currently running (e.g., "get_portfolio_positions…"). When the tool call completes, the chip disappears
  • Stop button: Allows cancelling a running response stream mid-generation

3. Slash commands

The following slash commands are recognised by the AI and trigger specific tool calls automatically. Any freeform text is also accepted — the AI handles natural language questions about investments, Thai SET stocks, and trading strategy.

4. AI tools

The AI can autonomously decide to call these tools during a conversation. Each tool call is shown to the user as a status chip before the result is incorporated into the response.

ToolWhen triggeredWhat it does
get_portfolio_positionsQuestions about current holdings, P&L, exposureReads the Excel file and fetches live Yahoo Finance prices; returns position list with P&L
get_live_priceQuestions about the current price of a specific stockQueries Yahoo Finance quote API for a ticker; handles .BK suffix automatically
get_performance_summaryQuestions about historical performanceReads performance data grouped by stock and by time period from the Excel service
read_knowledge_docQuestions requiring trading knowledge or strategy contextReads Markdown files from /app/investment_agent/knowledge/ (e.g., Fibonacci rules, personal trading style)
run_analysis_scriptDeep analysis requestsRuns a Python script from /app/investment_agent/ as a subprocess; returns stdout

The AI can chain up to 6 tool calls per message before being forced to provide a final response. This allows it to, for example, fetch portfolio positions, check live prices for each, and read a knowledge document — all in one message.

5. Knowledge base

The AI's system prompt is built dynamically by reading files from /app/investment_agent/ (mounted read-only from the host via Docker volume). This allows the knowledge base to be updated on the host without rebuilding or restarting the containers.

FilePurpose
CLAUDE.mdMain agent persona — trading style, preferred strategies, communication preferences
knowledge/personal-trading-style.mdPersonal trading rules and risk management guidelines
knowledge/knowledge_fibo.mdFibonacci analysis knowledge and application rules
knowledge/technical-analysis.mdTechnical analysis principles and indicator interpretation
Other *.md filesAny additional documents added to the knowledge directory are automatically included
To teach the AI new trading rules or strategies, add a Markdown file to the knowledge directory on the host. No restart required — the system prompt is rebuilt on every API call.

6. Session management

  • Each browser tab gets a unique session_id (UUID) generated when the first message is sent
  • The session stores the last 20 messages server-side, in-memory per Uvicorn worker
  • Clicking New Session calls DELETE /api/v1/ai/copilot/session/{id} to clear the history and generates a fresh session ID in the UI
  • Sessions are not persisted — they are lost when the backend container restarts or when the connection lands on a different Uvicorn worker
  • The session context allows the AI to refer back to earlier messages in the conversation ("As I mentioned earlier..." or "Building on what we discussed about GULF...")

7. Language behaviour

The AI responds in English by default. If the user writes a message in Thai, the AI detects this and responds in Thai. Mixed-language conversations are handled gracefully — the AI will match the language of the most recent user message.
