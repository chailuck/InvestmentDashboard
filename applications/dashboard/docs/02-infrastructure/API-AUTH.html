API Spec — Authentication

Objective: Issue and rotate JWT tokens, authenticate users by email/password, blacklist tokens on logout, and provide a self-service password-reset flow. All tokens are HS256-signed JWTs. Access tokens expire in 30 minutes; refresh tokens in 7 days. The token blacklist is stored in Redis with auto-expiring TTLs.

POST /api/v1/auth/login

Request body

{ "email": "admin@example.com", "password": "ChangeMe123!" }

Response 200

{
  "access_token":  "eyJhbGciOiJIUzI1NiJ9...",   // 30-minute JWT
  "refresh_token": "eyJhbGciOiJIUzI1NiJ9...",  // 7-day JWT
  "expires_in":    1800,                          // seconds
  "user": {
    "id":        "3fa85f64-5717-4562-b3fc-2c963f66afa6",
    "email":     "admin@example.com",
    "name":      "Admin",
    "role":      "admin",                         // "admin" | "analyst" | "viewer"
    "createdAt": "2026-01-15T08:00:00+07:00"
  }
}

JWT payload structure

{
  "sub":   "3fa85f64-...",    // user UUID — used by all auth dependencies
  "role":  "admin",
  "email": "admin@example.com",
  "type":  "access",          // "access" | "refresh"
  "jti":   "unique-uuid",     // blacklist key on logout
  "iat":   1748000000,
  "exp":   1748001800
}
401 Unauthorized — wrong email or password. Generic message to prevent user enumeration: {"detail": "Invalid credentials"}
403 Forbidden — account exists but is_active = false: {"detail": "Account disabled"}
429 Too Many Requests — exceeded 5 attempts per minute from this IP.

POST /api/v1/auth/refresh

Request body

{ "refresh_token": "eyJhbGciOiJIUzI1NiJ9..." }

Response 200

Same shape as login — returns a new access token AND a new refresh token (token rotation). The old refresh token should be discarded.

{
  "access_token":  "eyJ...",
  "refresh_token": "eyJ...",   // NEW refresh token — replaces the submitted one
  "expires_in":    1800,
  "user": { ... }
}
401 Unauthorized — token expired, invalid signature, or wrong type (e.g. passing an access token): {"detail": "..."}
401 Unauthorized — user deleted or deactivated since token was issued.

POST /api/v1/auth/logout

Request

POST /api/v1/auth/logout
Authorization: Bearer <access_token>
# No body

Response 200

{ "message": "Logged out successfully" }

The JTI from the access token is stored in Redis as blacklist:{jti} with a TTL equal to the token's remaining lifetime. Once blacklisted, the token cannot be used even if not expired.

Calling logout with an already-invalid or expired token still returns 200 — the response is identical to a successful logout to avoid leaking information about token state.

PUT /api/v1/auth/me

Any authenticated user can update their own display name. Email and role changes require admin access (use PUT /users/{id} instead).

Request body

Authorization: Bearer <access_token>

{ "name": "Alice Smith" }

Response 200

Same shape as GET /auth/me with updated name.

POST /api/v1/auth/forgot-password

Request body

{ "email": "user@example.com" }

Response 200 — email not found (or inactive)

{ "message": "If that email is registered, a reset link has been sent." }

Response 200 — development mode only

{
  "message":     "Reset token generated (dev mode — token returned directly).",
  "reset_token": "Bx7kP2qL..."     // 32-byte URL-safe token; use in reset-password
}
The reset token is stored in Redis as pwd_reset:{token} → user_id with a 1-hour TTL. In production, this token would be emailed; in development (APP_ENV=development) it is returned directly in the response body so the frontend can use it without email infrastructure.
429 Too Many Requests — exceeded 3 requests per minute per IP.
The same 200 response is returned whether the email is found or not — this prevents attackers from enumerating registered emails.

POST /api/v1/auth/reset-password

Request body

{
  "token":        "Bx7kP2qL...",   // token from forgot-password
  "new_password": "NewPass456!"
}

Response 200

{ "message": "Password has been reset successfully" }

On success the Redis key is deleted — the token is single-use.

400 Bad Request — token not found in Redis (expired or already used): {"detail": "Invalid or expired reset token"}
400 Bad Request — user associated with the token has been deactivated.
