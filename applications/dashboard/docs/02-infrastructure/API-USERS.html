API Spec — Users

Objective: Provide full user lifecycle management for admins (create, update, activate/deactivate, force-reset passwords, delete) and limited self-service for non-admins (change own password). Every user has one of three roles: admin (full access), analyst (read/write their own data), viewer (read-only). Roles are enforced via the require_admin FastAPI dependency.

Endpoints at a Glance

MethodPathAuthPurpose
GET/api/v1/usersAdminPaginated, filtered user list
POST/api/v1/usersAdminCreate user with role
GET/api/v1/users/{id}Admin or selfGet one user by UUID
PUT/api/v1/users/{id}Admin or selfUpdate name (self) or any field (admin)
POST/api/v1/users/{id}/activateAdminRe-enable a deactivated account
POST/api/v1/users/{id}/deactivateAdminDisable an account (cannot deactivate self)
POST/api/v1/users/{id}/reset-passwordAdminForce-set any user's password
POST/api/v1/users/me/change-passwordBearerChange own password (requires current password)

GET /api/v1/users

Query parameters

ParamTypeDefaultDescription
pageint ≥ 11Page number
page_sizeint 1–10020Results per page
rolestring—Filter by role: admin | analyst | viewer
is_activebool—Filter by active/inactive status
searchstring—Case-insensitive ILIKE match on name or email

Response 200

{
  "users": [ { /* UserDetail */ }, ... ],
  "total": 42     // total count across all pages
}

POST /api/v1/users

Request body

{
  "email":     "bob@example.com",
  "name":      "Bob",
  "password":  "SecurePass123",    // bcrypt-hashed before storage
  "role":      "viewer",           // "admin" | "analyst" | "viewer"
  "is_active": true                // optional, default true
}

Response 201

Returns the full UserDetail object.

409 Conflict — email already registered: {"detail": "Email already registered"}

PUT /api/v1/users/{id}

Non-admins can only update their own name. Admins can update any field on any user.

Request body (all fields optional)

{
  "name":      "Robert",       // any authenticated user updating themselves
  "email":     "new@example.com",  // admin only
  "role":      "analyst",          // admin only
  "is_active": false               // admin only
}

Response 200

Returns the updated UserDetail object.

403 Forbidden — non-admin trying to update another user or trying to change role/email.
404 Not Found — user UUID does not exist.
409 Conflict — admin changing email to one already in use.

POST /api/v1/users/{id}/deactivate

POST /api/v1/users/9f4c3a21-.../deactivate
Authorization: Bearer <admin-token>
# No body

HTTP/1.1 200 OK
{ /* UserDetail with is_active: false */ }

POST /api/v1/users/me/change-password

  "new_password":     "NewPass456!"
