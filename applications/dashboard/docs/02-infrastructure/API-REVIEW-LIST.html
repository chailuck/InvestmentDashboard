API — Weekly Review List

Documents all endpoints for the Weekly Review List feature. This API allows analysts to record and annotate trade activity and open positions on a per-ISO-week basis.

Base URL prefix/api/v1/review-list
AuthenticationAll endpoints require Authorization: Bearer <JWT>. Tokens are issued by POST /api/v1/auth/login.
OpenAPI tagReview List
Database tablesweekly_reviews, weekly_review_items — see DATABASE.html
ORM modelsbackend/app/models/weekly_review.py
Router filebackend/app/api/v1/review_list.py

1. Data models

ReviewSummary

Compact view of a review header — returned by the list and current-week endpoints. Does not include items.

{
  "id":          "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "week_start":  "2026-06-08",
  "week_end":    "2026-06-14",
  "name":        "Week 24 (08 Jun–14 Jun 2026)",
  "trade_count": 3,    // number of TRADE items
  "hold_count":  2,    // number of HOLD items
  "buy_count":   2,    // TRADE items with a buy leg this week
  "sell_count":  1,    // TRADE items with a sell leg this week
  "created_at":  "2026-06-08T09:00:00+00:00",
  "updated_at":  "2026-06-13T14:22:00+00:00"
}

ReviewDetail

Full review including sorted items and open position suggestions — returned by GET /{review_id}.

{
  "id":          "uuid",
  "week_start":  "2026-06-08",
  "week_end":    "2026-06-14",
  "name":        "Week 24 (08 Jun–14 Jun 2026)",
  "notes":       null,
  "created_at":  "2026-06-08T09:00:00+00:00",
  "updated_at":  "2026-06-13T14:22:00+00:00",
  "trade_items": [ /* ReviewItem objects — see below */ ],
  "hold_items":  [ /* ReviewItem objects */ ],
  "open_suggestions": [
    {
      "id":            "uuid",
      "symbol":        "ADVANC",
      "entry_date":    "2026-05-15",
      "entry_price":   188.00,
      "position_size": 500,
      "direction":     "long"
    }
  ]
}
trade_items are sorted by in-week activity date: sell_date if the sell date falls in the review week, else buy_date if the buy date falls in the review week, else buy_date. hold_items are sorted alphabetically by symbol. open_suggestions lists active portfolio_positions_db positions that are not already present in trade_items, provided as candidates for the analyst to add as HOLD items.

ReviewItem

One position entry inside a review. The same shape is used in both trade_items and hold_items arrays and is returned by single-item POST/PATCH endpoints.

{
  "id":                 "uuid",
  "review_id":          "uuid",
  "symbol":             "AOT",
  "item_type":          "TRADE",       // "TRADE" | "HOLD"

  // Buy leg — null if position was not entered this week
  "buy_date":           "2026-06-09",
  "buy_price":          52.50,
  "buy_size":           1000,

  // Sell leg — null if position was not exited this week
  "sell_date":          null,
  "sell_price":         null,
  "sell_size":          null,

  // Week price snapshot (populated by /refresh-prices)
  "week_open_price":    51.00,
  "week_close_price":   53.25,
  "week_change_pct":    4.41,          // COMPUTED — not stored in DB

  // Analyst annotations
  "buy_reason":         "Strong breakout above resistance",
  "buy_feeling":        4,             // 1–5 or null
  "sell_reason":        null,
  "sell_feeling":       null,

  "source_position_id": "uuid-or-null",
  "sort_order":         0,
  "updated_at":         "2026-06-13T10:00:00+00:00"
}
week_change_pct is a computed field: (week_close_price − week_open_price) / week_open_price × 100. It is calculated by the API response serialiser and does not exist as a database column.

ReviewItemIn

Request body schema for creating or bulk-replacing items.

{
  "symbol":             "AOT",
  "item_type":          "TRADE",       // required: "TRADE" | "HOLD"
  "buy_date":           "2026-06-09",  // optional
  "buy_price":          52.50,         // optional
  "buy_size":           1000,          // optional
  "sell_date":          null,          // optional
  "sell_price":         null,          // optional
  "sell_size":          null,          // optional
  "buy_reason":         "Breakout",    // optional
  "buy_feeling":        4,             // optional: 1–5 or null
  "sell_reason":        null,          // optional
  "sell_feeling":       null,          // optional: 1–5 or null
  "source_position_id": null,          // optional UUID
  "sort_order":         0              // optional, default 0
}

2. Endpoints

GET /api/v1/review-list/current-week Get or create this week's review

Returns the review for the current ISO week (Monday–Sunday). If a review does not exist yet for the authenticated user, one is created automatically before responding. This is the primary entry point for the weekly review UI.

Response 200 — ReviewSummary

{
  "id":          "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "week_start":  "2026-06-08",
  "week_end":    "2026-06-14",
  "name":        "Week 24 (08 Jun–14 Jun 2026)",
  "trade_count": 3,
  "hold_count":  2,
  "buy_count":   2,
  "sell_count":  1,
  "created_at":  "2026-06-08T09:00:00+00:00",
  "updated_at":  "2026-06-13T14:22:00+00:00"
}
This endpoint is idempotent — calling it multiple times is safe. The auto-created review is given a name in the format "Week {N} ({DD Mon}–{DD Mon YYYY})" where N is the ISO week number.

POST /api/v1/review-list/ Create a review manually

Creates a new weekly review for a given week_start date. The date must be a Monday (the API does not auto-adjust). If a review already exists for that week, a 409 is returned.

Request body

{
  "week_start": "2026-06-01",   // required — must be a Monday
  "name":       "My custom name" // optional — auto-generated if omitted
}

Response 201

{
  "id":         "uuid",
  "week_start": "2026-06-01",
  "name":       "Week 23 (01 Jun–07 Jun 2026)"
}

Response 409 — conflict

{ "detail": "A review already exists for the week starting 2026-06-01." }
If name is omitted the server auto-generates it from the ISO week number and date range. The name can be changed later with PUT /{review_id}.

DELETE /api/v1/review-list/{review_id} Delete a review

Permanently deletes the review and all its items. This action is irreversible. The cascade is handled at the database level (ON DELETE CASCADE on weekly_review_items.review_id).

Path parameters

ParameterTypeNotes
review_idUUIDID of the review to delete

Response 204 — No Content

Empty body. Indicates successful deletion.

Response 404

{ "detail": "Review not found." }
POST /api/v1/review-list/{review_id}/sync Auto-populate TRADE items from portfolio DB

Scans portfolio_positions_db for positions where entry_date OR exit_date falls within the review week. Matching positions are upserted as TRADE items keyed by source_position_id. Existing manually-entered items (where source_position_id IS NULL) are not affected.

Path parameters

ParameterTypeNotes
review_idUUIDID of the review to sync

Response 200

{ "added": 2, "updated": 1 }
FieldNotes
addedNumber of new TRADE items inserted
updatedNumber of existing TRADE items (by source_position_id) updated with fresh data from the portfolio DB
Run this after recording trades in the portfolio DB to pull them into the review automatically. Safe to run multiple times — it is idempotent.

Response 404

{ "detail": "Review not found." }
POST /api/v1/review-list/{review_id}/refresh-prices Fetch week open/close prices from Yahoo Finance

Fetches Monday-open and Friday-close prices from Yahoo Finance for every unique symbol in the review. All symbols are treated as Thai SET equities — the .BK suffix is appended automatically before querying yfinance. For reviews covering an in-progress week, the Friday close falls back to the latest available close price.

Path parameters

ParameterTypeNotes
review_idUUIDID of the review to refresh prices for

Response 200

{
  "updated": 5,
  "symbols": [
    { "symbol": "AOT",    "week_open_price": 51.00,  "week_close_price": 53.25 },
    { "symbol": "ADVANC", "week_open_price": 188.00, "week_close_price": 191.50 },
    { "symbol": "CPALL",  "week_open_price": 62.50,  "week_close_price": 63.00 },
    { "symbol": "SCB",    "week_open_price": 104.00, "week_close_price": 102.50 },
    { "symbol": "PTT",    "week_open_price": 34.00,  "week_close_price": 34.75 }
  ]
}
This call makes external HTTP requests to Yahoo Finance and may take several seconds for reviews with many symbols. The operation updates week_open_price and week_close_price in weekly_review_items for all items sharing the same symbol in this review.

Response 404

{ "detail": "Review not found." }

Replaces ALL items in the review atomically (delete then re-insert within a single transaction). Use this endpoint when the frontend manages the full item list locally and pushes the complete state on save. Any items not included in the request body are permanently deleted.

PATCH /api/v1/review-list/{review_id}/items/{item_id} Update annotations only (auto-save)

Partially updates an item's annotation fields: buy_reason, buy_feeling, sell_reason, sell_feeling. Only fields present in the JSON body are written — absent fields are left unchanged (true PATCH semantics). Designed for auto-save use cases where the frontend sends the current annotation state on every keypress or on blur.

Path parameters

ParameterTypeNotes
review_idUUIDID of the parent review
item_idUUIDID of the item to update

Request body — all fields optional

{
  "buy_reason":   "Good momentum setup",
  "buy_feeling":  4,
  "sell_reason":  "Hit take-profit target",
  "sell_feeling": 5
}
FieldTypeNotes
buy_reasonstring | nullFreeform rationale for the buy decision
buy_feelinginteger 1–5 | nullEmotional rating for the buy decision; null = unrated
sell_reasonstring | nullFreeform rationale for the sell decision
sell_feelinginteger 1–5 | nullEmotional rating for the sell decision; null = unrated

Response 200 — full ReviewItem

Returns the complete updated item (same shape as the ReviewItem schema above).

Response 400 — validation error

{ "detail": "sell_feeling must be an integer between 1 and 5, or null." }

Response 404

{ "detail": "Item not found or does not belong to this review." }
This endpoint only updates the four annotation fields. To update trade fields (prices, dates, sizes), use the bulk PUT /{review_id}/items endpoint.

3. Error responses

All error responses follow the standard FastAPI error envelope: { "detail": "..." }. The table below summarises the conditions under which each HTTP status code is returned.

StatusCondition
400 Bad Request
  • item_type is not TRADE or HOLD
  • buy_feeling or sell_feeling is outside the 1–5 range
  • Malformed request body (missing required fields, wrong types)
401 Unauthorized Missing, expired, or invalid Bearer JWT in the Authorization header
404 Not Found Review or item not found, or the resource belongs to a different user
409 Conflict A review already exists for the given week_start date (POST / only)

4. Feeling rating scale

Both buy_feeling and sell_feeling use the same 5-point scale. Null means the field has not been rated yet.

ValueLabelIntended meaning
1Very BadThe decision was clearly wrong or emotional in hindsight
2BadThe decision had significant flaws
3ModerateThe decision was acceptable but could have been better
4GoodThe decision followed the plan with good discipline
5Very GoodThe decision was well-timed, well-reasoned, and well-executed
nullUnratedThe field has not yet been evaluated
The feeling rating reflects the quality of the decision, not the outcome. A well-executed buy that resulted in a loss can still be rated 5 if it followed the trading plan. An impulsive buy that happened to profit should be rated 1 or 2.
