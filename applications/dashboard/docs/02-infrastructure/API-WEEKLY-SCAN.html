API Spec — Weekly Scan

Objective: Manage the weekly manual stock scan workflow — from maintaining symbol lists through creating scan snapshots, evaluating symbols with colour marks and strategies, fetching weekly prices (Monday open / Friday close), and viewing DR-adjusted prices. All data is private to the requesting user. Scans are named with a date pattern (WEEKLY_SCAN_DD_MM_YYYY) used to derive the price dates automatically.

Symbol Lists

GET /api/v1/weekly-scan/symbol-lists

Returns all symbol lists for the user ordered by sort_order. If the user has no lists but has a legacy config, a "Default" list is auto-seeded from the config on first access.
[
  {
    "id":         "list-uuid-...",
    "name":       "SET50",
    "market":     "SET",          // "SET" | "US" | "CRYPTO" | "HK" | "OTHER"
    "is_dr":      false,          // true = this list contains parent symbols; show DR prices
    "symbols":    ["ADVANC", "AOT", "KBANK", "GULF"],
    "sort_order": 0,
    "updated_at": "2026-05-30T10:00:00+07:00"
  }
]

POST /api/v1/weekly-scan/symbol-lists

// Request
{
  "name":    "DRs",
  "market":  "CRYPTO",    // optional, default "SET"
  "is_dr":   true,        // optional, default false
  "symbols": ["BTC-USD", "ETH-USD"]
}

// Response 200
{ /* UserSymbolList object */ }

PUT /api/v1/weekly-scan/symbol-lists/{list_id}

// All fields optional — only provided fields are updated
{
  "name":       "My DRs",
  "symbols":    ["BTC-USD", "ETH-USD", "GC=F"],
  "sort_order": 2
}
404 Not Found — list not found or not owned by user.

POST /api/v1/weekly-scan/scans

Creates a scan and populates it with all symbols from the user's current symbol lists. Items inherit list name and market. If no lists exist, falls back to the legacy symbol config (or SET50 defaults).

{ "name": "WEEKLY_SCAN_07_06_2026" }

// Response 200

  "id":         "scan-uuid-...",
  "name":       "WEEKLY_SCAN_07_06_2026",
  "created_at": "2026-06-03T08:00:00+07:00"
}

PUT /api/v1/weekly-scan/scans/{scan_id}/items/{symbol}

Partial update — only fields present in the request body are written. Used after every click in the scan table (colour picker, strategy dropdown, numeric field edit).
// Request — any subset of these fields
{
  "color_mark": "CYAN",
  "strategy":   "BUY ON DIP",
  "buy_price":  63.0,
  "size":       300,
  "tp":         70.0,
  "sl":         58.0,
  "remark":     "Watch for close above 63"
}

// Response 200 — full updated item
{ /* WeeklyScanItem */ }
404 Not Found — scan not found: {"detail": "Scan not found"}
404 Not Found — symbol not in scan: {"detail": "GULF not in scan"}

GET /api/v1/weekly-scan/scans/{scan_id}/week-prices

Fetches Monday-open and Friday-close prices for every symbol in the scan. Week dates are parsed from the scan name pattern WEEKLY_SCAN_DD_MM_YYYY. DR symbols (mapped via the dr_mappings table) additionally return parent asset prices and estimated THB prices. All yfinance calls run concurrently with a semaphore of 5.

Response 200

{
  "mon_date": "2026-06-02",
  "fri_date": "2026-06-06",
  "usd_thb":  34.12,          // USD/THB rate (null if no DR symbols in scan)

  "prices": {
    "GULF": {
      "mon":        63.50,
      "fri":        65.25,
      "current":    65.00,
      "change_abs": -0.25,
      "change_pct": -0.38
    },
    "BTCUSD-DR": {
      "mon":           67000.0,    // parent BTC-USD price (USD)
      "fri":           68200.0,
      "current":       68000.0,
      "parent_mon":    67000.0,
      "parent_fri":    68200.0,
      "dr_mon_thb":    2283.52,    // parent_mon / ratio × usd_thb
      "dr_fri_thb":    2324.36,
      "dr_current_thb": 2316.53,
      "parent_symbol": "BTC-USD",
      "ratio":         1000.0
    },
    "ADVANC": {
      "mon":  null,               // null = no data from Yahoo Finance for that date
      "fri":  null,
      "current": 217.50,
      "change_abs": 1.50,
      "change_pct": 0.69
    }
  }
}
If the scan name does not match the WEEKLY_SCAN_DD_MM_YYYY pattern, mon_date and fri_date are null and all prices will be null. The response body is still returned (HTTP 200) so the UI can show an informative message.
