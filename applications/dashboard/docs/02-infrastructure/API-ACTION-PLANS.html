API Spec — Action Plans

Objective: Manage investment action plans — structured documents that capture planned trades or portfolio adjustments. Two plan types exist: purchase (stocks to buy, with buy-price, TP, SL, strategy) and portfolio (existing holdings with entry price and target adjustments). Plans are private to the creating user. The PUT endpoint performs a full replace of all items (delete-all + bulk insert) in a single transaction.

GET /api/v1/action-plans/suggest-name

Returns a name that does not yet exist for this user + plan_type. Pattern: YYYY-MM-DD, then YYYY-MM-DD-01, YYYY-MM-DD-02, …

Query parameters

ParamRequiredValues
plan_typeYespurchase | portfolio

Sample

GET /api/v1/action-plans/suggest-name?plan_type=purchase
Authorization: Bearer eyJ...

{ "name": "2026-06-03" }      // or "2026-06-03-01" if today already exists

GET /api/v1/action-plans/stock-price

Fetches the latest closing price for a Thai SET stock via Yahoo Finance. The backend tries {SYMBOL}.BK first, then falls back to the bare symbol, so both Thai and non-Thai tickers can be queried through the same endpoint. Used by the purchase plan editor both on individual stock-field blur and by the Refresh All Prices batch operation.

Authentication

Bearer JWT required. Returns 401 if the token is missing or invalid.

Query parameters

ParamRequiredTypeExample
symbolYesstringBH, GULF, ADVANC

Response 200 — success

FieldTypeDescription
symbolstringThe symbol exactly as submitted in the query parameter
tickerstringThe Yahoo Finance ticker that returned data (GULF.BK or bare GULF)
pricenumberLatest available closing price (float, in the stock's native currency — THB for SET stocks)
GET /api/v1/action-plans/stock-price?symbol=GULF
Authorization: Bearer eyJ...

HTTP/1.1 200 OK
{ "symbol": "GULF", "ticker": "GULF.BK", "price": 64.25 }
401 Unauthorized — Token missing or expired: {"detail": "Not authenticated"}
404 Not Found — Yahoo Finance returns no data for either {SYMBOL}.BK or bare {SYMBOL}: {"detail": "Price not found for GULF"}
422 Unprocessable Entity — symbol query parameter is absent or empty: standard Pydantic validation error array.

Usage in Refresh All Prices

The frontend's refreshAllPrices function calls this endpoint once per non-empty row in parallel (Promise.all with per-item .catch). A 404 or network error for one symbol does not abort the batch — it is counted as a failed symbol in the refresh result notification. No automatic save occurs after a batch refresh; the user must click Save explicitly.

PUT /api/v1/action-plans/{id}

Full replace: all fields are optional. If purchase_items is provided, ALL existing items are deleted and replaced with the new list. Items are re-indexed by their position in the array (sort_order = 0, 1, 2, …). Omit the purchase_items key entirely to leave items unchanged.

Request body

{
  "name":       "2026-06-03",           // optional
  "notes":      "Updated analysis.",    // optional; null clears the field
  "set_analysis": null,
  "ai_recommend": null,

  "purchase_items": [                   // omit key to keep existing items
    {
      "sort_order":    0,               // ignored — position in array is used
      "stock":         "GULF",
      "current_price": 64.25,
      "size":          500,
      "buy_price":     63.50,
      "tp":            72.00,
      "sl":            59.00,
      "strategy":      "BREAK OUT",
      "reason":        "Weekly breakout",
      "triggered":     false
    }
  ]
}

Response 200

{ "status": "ok" }
The id field in request items is ignored — it is provided by the server. Each PUT generates new item UUIDs for all items in the list.
404 Not Found — plan not found or not owned by the requesting user.

DELETE /api/v1/action-plans/{id}

Hard-deletes the plan and all its items (cascade). Returns HTTP 204 No Content.

404 Not Found — plan not found or not owned by requesting user.

POST /api/v1/action-plans/{id}/duplicate

Creates a deep copy of a plan — header + all items — under a new name. The duplicate belongs to the same user. Useful for using last week's plan as a template for this week.

Query parameters

ParamRequiredExample
new_nameYes2026-06-10

Response 201

{
  "id":        "new-plan-uuid-...",
  "name":      "2026-06-10",
  "plan_type": "purchase"
}
404 Not Found — source plan not found.
