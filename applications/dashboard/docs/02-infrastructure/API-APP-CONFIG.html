API Spec — App Configuration

Objective: Read and write runtime application settings stored in app_config.json inside the container. This allows operators to change Excel file paths and knowledge-base directories without rebuilding the Docker image or restarting the backend. Settings are merged into the JSON file on write — only provided keys are updated. All authenticated users can read config; all authenticated users can write config (not admin-only).

Endpoints at a Glance

MethodPathAuthPurpose
GET/api/v1/app-configBearerReturn current configuration
PUT/api/v1/app-configBearerUpdate one or more config keys
POST/api/v1/app-config/test-pathBearerCheck if a container path is a readable file

Config Key Reference

KeyDefault (from env)Purpose
excel_source_pathINVESTMENT_EXCEL_SOURCE_PATH or INVESTMENT_EXCEL_PATHRead-only master Excel file (Docker volume mount)
excel_working_pathINVESTMENT_EXCEL_PATHWritable working copy that the portfolio service reads
analysis_log_path/app/investment_agent/analysis_logDirectory containing HTML/MD analysis log files
fibo_chart_path/app/investment_agent/charts/FiboDirectory containing Fibonacci chart image files

GET /api/v1/app-config

Response 200

{
  "excel_source_path":  "/app/investment_data/Investment tracking.xlsx",
  "excel_working_path": "/app/uploads/portfolio.xlsx",
  "analysis_log_path":  "/app/investment_agent/analysis_log",
  "fibo_chart_path":    "/app/investment_agent/charts/Fibo"
}

Values fall back to environment-variable defaults if the key has never been explicitly saved to app_config.json.

PUT /api/v1/app-config

Merges the provided keys into the existing app_config.json. Only keys that are explicitly provided (and non-null) are written — other keys are left unchanged. The portfolio Excel service reads the updated paths on the very next request (no restart needed).

Request body (all fields optional)

{
  "excel_source_path":  "/app/investment_data/MyPortfolio.xlsx",
  "excel_working_path": "/app/uploads/working.xlsx"
}

Response 200

{
  "excel_source_path":  "/app/investment_data/MyPortfolio.xlsx",
  "excel_working_path": "/app/uploads/working.xlsx",
  "analysis_log_path":  "/app/investment_agent/analysis_log",
  "fibo_chart_path":    "/app/investment_agent/charts/Fibo"
}

The full merged config is returned, not just the updated keys.

After changing excel_source_path, call POST /portfolio-tracker/refresh to copy the new source file to the working path and clear all worker caches.

POST /api/v1/app-config/test-path

Checks whether the given path exists and is a readable file inside the container. Used by the Settings UI "Test" button before saving a new Excel path — prevents misconfigured paths from breaking data loading.

Request body

{ "excel_source_path": "/app/investment_data/Investment tracking.xlsx" }

Only excel_source_path is evaluated; excel_working_path is ignored by this endpoint.

Response 200 — file exists and readable

{ "ok": true,  "message": "File found (142.3 KB) — ready to use." }

Response 200 — file not found

{ "ok": false, "message": "File not found: /app/investment_data/wrong.xlsx" }

Response 200 — path is a directory, not a file

{ "ok": false, "message": "Path exists but is not a file: /app/investment_data" }

Response 200 — no path provided

{ "ok": false, "message": "No path provided." }
This endpoint always returns HTTP 200 — the ok boolean in the body indicates success or failure. This makes it easy to show inline feedback in the UI without catching HTTP errors.
