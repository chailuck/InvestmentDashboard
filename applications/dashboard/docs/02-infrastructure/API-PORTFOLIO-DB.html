API Spec — Portfolio DB

Objective: Provide full CRUD for portfolio positions stored in PostgreSQL, as an alternative to the Excel-backed tracker. Supports full and partial sells (with parent-child record model), sell undo, and mode switching per user. Live prices are fetched from Yahoo Finance for active positions. The portfolio tracker endpoints (/portfolio-tracker/*) automatically route to these DB helpers when the user's portfolio_mode is db.

PUT /api/v1/portfolio-db/positions/{id}

Full replace — same body as POST. All fields that are omitted default to null. Returns updated position object.

404 Not Found — position UUID not found or not owned by user.

DELETE /api/v1/portfolio-db/positions/{id}

Hard delete. Returns HTTP 204 No Content.

404 Not Found — position UUID not found or not owned by user.

POST /api/v1/portfolio-db/positions/{id}/sell

Sell all or part of an active position.
  • Full sell (quantity = position_size): closes the position in-place.
  • Partial sell (quantity < position_size): creates a new closed child record with parent_id pointing to the original, and reduces the parent's position_size.

Request body

{
  "quantity":   200,              // shares to sell — must be > 0 and ≤ position_size
  "exit_price": 68.50,
  "exit_date":  "2026-06-02",
  "remarks":    "TP hit"          // optional
}

Response 201 — full sell

{
  "type":     "full",
  "position": { /* closed position object */ }
}

Response 201 — partial sell

{
  "type":      "partial",
  "remaining": { /* active parent — positionSize reduced */ },
  "sold":      { /* new closed child — parentId set */ }
}
400 Bad Request — position is not active: {"detail": "Position is not active"}
400 Bad Request — quantity is zero or negative: {"detail": "Sell quantity must be positive"}
400 Bad Request — selling more than owned: {"detail": "Cannot sell more than 300 shares"}
404 Not Found — position not found.

POST /api/v1/portfolio-db/positions/{id}/undo-sell

Deletes the most recently created child sell record and restores the parent's position_size. Only the last partial sell can be undone — calling undo twice will undo the previous sell as well.
// Request: no body

// Response 200
{
  "status":            "ok",
  "restored_quantity": 200,
  "position":          { /* updated parent position */ }
}
404 Not Found — no child sell record exists for this position: {"detail": "No sell record found to undo"}
