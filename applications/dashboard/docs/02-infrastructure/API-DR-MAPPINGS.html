API Spec — DR Mappings

Objective: Manage the global mapping table that links Thai-listed Depositary Receipts (DRs) to their underlying parent assets. This mapping is used by the Weekly Scan price engine to look up the parent asset price (e.g. BTC-USD) and compute the estimated DR price in THB using the unit ratio and the live USD/THB exchange rate. Read access is open to all authenticated users; writes require admin role. DR symbols are stored and matched case-insensitively (always uppercased).

Price Estimation Formula

DR estimated price (฿) = parent_price_USD ÷ ratio × USD_THB_rate

// Example: BTCUSD-DR with ratio = 1000
//   BTC-USD = $60,000  |  USD/THB = 34.0
//   DR price = 60,000 ÷ 1,000 × 34 = ฿2,040 per DR unit

PUT /api/v1/dr-mappings/{id}

Updates one or more fields. The description field is always auto-regenerated from the current parent_symbol and ratio after any update — it cannot be set manually.

Request body (all fields optional)

{
  "parent_symbol": "ETH-USD",    // max 30 chars; uppercased on save
  "parent_market": "CRYPTO",     // max 20 chars
  "ratio":         150.0,        // must be > 0
  "is_active":     false
}

Bulk Import / Export (Frontend)

The Settings → DR Mappings page supports file-based bulk operations that call the API in a loop. The file format is pipe-delimited text:

DR_SYMBOL|PARENT_SYMBOL|MARKET|RATIO    ← header line (skipped on import)
BTCUSD-DR|BTC-USD|CRYPTO|1000
GOLUSD-DR|GC=F|COMMODITY|10
ETHUSD-DR|ETH-USD|CRYPTO|100

Import calls POST /dr-mappings for each line. Lines that result in a 409 (duplicate) are counted as skipped, not errors. All other errors also count as skipped.
