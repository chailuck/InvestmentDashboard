API Spec — AI Copilot

Objective: Stream an agentic AI conversation powered by the Anthropic API (Claude). The copilot has access to live portfolio data, market prices, knowledge-base documents, and analysis scripts via five tools. Responses stream word-by-word as Server-Sent Events (SSE). Each session maintains up to 20 messages of conversation history stored in the backend process memory. Requires ANTHROPIC_API_KEY to be configured.

Endpoints at a Glance

MethodPathAuthPurpose
POST/api/v1/ai/copilot/streamBearerSend a message — stream SSE response

POST /api/v1/ai/copilot/stream

Sends a user message and streams the assistant response as SSE events. If no session_id is provided, a new UUID is generated and returned in the X-Session-Id response header. Pass the same session_id in subsequent requests to continue the conversation.

Request body

{
  "message":    "/portList",             // user message text
  "session_id": "3f9a1c2e-..."           // optional; omit to start a new session
}

Response headers

Content-Type:  text/event-stream
Cache-Control: no-cache
X-Accel-Buffering: no
X-Session-Id:  3f9a1c2e-...             // use this in the next request to continue

SSE event stream format

Each line is data: {JSON}\n\n. Events arrive in this order for a typical tool-using response:

data: {"type": "tool_start", "name": "get_portfolio_positions"}

data: {"type": "tool_end", "name": "get_portfolio_positions"}

data: {"type": "text", "content": "Here "}
data: {"type": "text", "content": "are "}
data: {"type": "text", "content": "your "}
data: {"type": "text", "content": "5 "}
data: {"type": "text", "content": "active "}
data: {"type": "text", "content": "positions:\n\n"}
data: {"type": "text", "content": "| Symbol ..."}

data: {"type": "done", "session_id": "3f9a1c2e-..."}

All SSE event types

typeAdditional fieldsDescription
textcontent: stringOne word (or token) of the assistant's reply — concatenate to build the full message
tool_startname: stringA tool call has started — show a "thinking…" indicator
tool_endname: stringTool call completed — hide the indicator
errorcontent: stringAPI error or missing key — stream ends after this event
donesession_id: stringStream complete — always the last event

Slash-command → tool mapping

CommandTool(s) invoked
/portListget_portfolio_positions
/portPlan, /portActionget_portfolio_positions + get_performance_summary
/portHistget_performance_summary
/analyze TICKERget_live_price then read_knowledge_doc
Error event — ANTHROPIC_API_KEY not set: {"type":"error","content":"Anthropic API key not configured. Set ANTHROPIC_API_KEY in backend/.env and restart the backend."}
Error event — Anthropic API returned an error (rate limit, bad key, etc.): {"type":"error","content":"..."}
The AI Copilot is an agentic loop — it may call tools multiple times before producing a final text response. The maximum number of tool-call iterations per request is 6. Session history is in-memory per Uvicorn worker; if the server has multiple workers, the same session_id may hit a different worker with no prior context.

DELETE /api/v1/ai/copilot/session/{session_id}

Clears the conversation history stored in the server-side session dictionary. Call this to start a fresh conversation without creating a new session_id, or when the user clicks "Clear chat".

Sample

DELETE /api/v1/ai/copilot/session/3f9a1c2e-...
Authorization: Bearer eyJ...

HTTP/1.1 200 OK
{ "status": "cleared", "session_id": "3f9a1c2e-..." }

Deleting a session_id that does not exist (or was already cleared) returns the same 200 response — this is idempotent.
