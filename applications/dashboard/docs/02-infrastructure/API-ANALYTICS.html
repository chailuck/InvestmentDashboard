API Spec — Analytics

Objective: Provide symbol lookup, OHLCV chart data with technical indicators (RSI, Stochastic, VRVP), and per-user supplementary content — analysis logs (HTML/MD files from the investment agent knowledge base), Fibonacci chart images (from the knowledge base), and personal text notes stored in PostgreSQL. All data is fetched on demand with no caching at the API layer. Symbol inputs are validated against a strict regex and file-system access is path-traversal-hardened.

Endpoints at a Glance

GET/api/v1/analytics/chartBearerOHLCV + RSI + Stochastic + VRVP

GET /api/v1/analytics/search

Quick symbol lookup — tries multiple Yahoo Finance ticker variants in order and returns the first with price data. Used by the search form to validate a symbol before opening the chart modal.

Query parameters

ParamRequiredDefaultValues
qYes (min 1 char)—Symbol, e.g. GULF, BTC, GOLD
asset_typeNoSETSET | CRYPTO | DR | US | HK | OTHER

Ticker resolution order by asset_type

asset_typeTries in order
SETSYMBOL.BK → SYMBOL
CRYPTOSYMBOL-USD → SYMBOL-USDT → SYMBOL
DRSYMBOL.BK → SYMBOL
HK0001.HK (zero-padded 4 digits) → SYMBOL.HK → SYMBOL
US / OTHERSYMBOL (bare)

Sample — found

GET /api/v1/analytics/search?q=GULF&asset_type=SET
Authorization: Bearer eyJ...

{
  "symbol":     "GULF",
  "ticker":     "GULF.BK",
  "asset_type": "SET",
  "exchange":   "SET",
  "name":       "Gulf Energy Development Public Company Limited",
  "price":      64.25,
  "change_pct": 1.18,
  "found":      true
}

Sample — not found

GET /api/v1/analytics/search?q=INVALID&asset_type=SET

{ "symbol": "INVALID", "found": false }

GET /api/v1/analytics/chart

Returns the full dataset for the ECharts chart modal: candlestick OHLCV, volume bars, RSI(14), Stochastic(9,3,3), and VRVP(24 bins). Fetches up to 2.5 years of daily/weekly data or 2 years of hourly data. All yfinance calls run in a thread pool to avoid blocking the async event loop.

Query parameters

ParamRequiredDefaultValues
symbolYes—Trading symbol
asset_typeNoSETSame as search
intervalNo1d1h | 4h | 1d | 1wk

Response 200 (abbreviated)

{
  "symbol":   "GULF",
  "ticker":   "GULF.BK",
  "exchange": "SET",

  "candles": [
    { "time": "2026-05-30", "open": 63.25, "high": 64.50, "low": 62.75, "close": 64.00 }
  ],

  "volume": [
    { "time": "2026-05-30", "value": 12500000, "color": "rgba(34,197,94,0.5)" }
  ],                                       // green = up candle, red = down candle

  "rsi": [
    { "time": "2026-05-30", "value": 58.42 }
  ],                                       // NaN values omitted

  "stoch_k": [{ "time": "2026-05-30", "value": 72.34 }],  // %K smoothed (9,3)
  "stoch_d": [{ "time": "2026-05-30", "value": 68.12 }],  // %D signal line

  "vrvp": [
    { "price_low": 55.00, "price_high": 55.46, "volume": 45200000 }
    // ... 24 bins total spanning the full price range of loaded history
  ]
}
404 Not Found — no price data found for any ticker variant for this symbol + asset_type.

GET /api/v1/analytics/analysis-log

Searches the analysis_log_path directory for files matching *{SYMBOL}*.html or *{SYMBOL}*.md and returns the most recently modified file. All path results are validated to be inside the configured base directory before reading (path-traversal guard).

Query parameters

ParamRequiredValidation
symbolYesMust match ^[A-Z0-9.\-]{1,20}$ — 400 on failure

Sample — file found

GET /api/v1/analytics/analysis-log?symbol=GULF
Authorization: Bearer eyJ...

{
  "found":     true,
  "content":   "<!DOCTYPE html><html>...</html>",
  "filename":  "GULF_analysis_2026-05-30.html",
  "file_type": "html"          // "html" | "md"
}

Sample — not found

{ "found": false, "content": null, "filename": null, "file_type": null }
400 Bad Request — symbol contains invalid characters: {"detail": "Invalid symbol: must be 1-20 alphanumeric characters, dots, or hyphens"}

GET /api/v1/analytics/fibo-chart

Searches the fibo_chart_path directory for image files matching the symbol. Returns the image as a Base64 data URI so the frontend can embed it directly without a second request.

Query parameters

ParamRequiredAccepted formats
symbolYesSame validation as analysis-log; matches .png .jpg .jpeg .webp

Sample — found

{
  "found":    true,
  "image":    "data:image/png;base64,iVBORw0KGgoAAAANSUhEUg...",
  "filename": "GULF_fibo_2026-05.png"
}
// Not found: { "found": false, "image": null, "filename": null }

PUT /api/v1/analytics/note

PostgreSQL upsert on (user_id, symbol) constraint. Concurrent upserts are safe. Saving an empty string stores a blank note but does not delete the record.
// Request
{ "symbol": "GULF", "asset_type": "SET", "note": "Watch RSI divergence on weekly." }

// Response 200
{ "status": "ok" }
